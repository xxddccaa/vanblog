# v1.6.2

`v1.6.2` 是一次文章分类模型升级版本：文章从“单分类”扩展为“多分类 / 多专栏”，同时保留旧 `category` 字段与旧接口兼容。

如果只用一句话概括：

> `v1.6.2` 让一篇文章可以同时归属多个分类 / 专栏，并让后台新建、编辑、导入、前台分类页、RSS、Sitemap 与搜索索引都识别这些分类。

## 本版摘要

这一版主要集中在三件事：

- 文章新增 `categories: string[]` 多分类字段，旧 `category` 字段继续作为第一个分类兼容保留
- 后台文章表单支持搜索、输入并自动创建分类 / 专栏，一篇文章可选择多个分类
- 修复多分类改造过程中带出的颜色回归，恢复前台分类链接与后台主按钮在暗色模式下的蓝色主色

## 相比 v1.6.1 的变化

如果你已经在使用 `v1.6.1`，这一版最直接的变化是：

- 新建文章和修改文章时，“分类 / 专栏”支持多选
- 可以直接在文章表单里输入不存在的分类名，提交时自动创建分类
- 多分类文章会出现在每个所属分类页
- 分类搜索、分类归档、分类统计、RSS、Sitemap、Search Index 都会识别 `categories`
- 旧接口只传 `category` 仍然可用
- 后台暗色模式下“新建文章”等主按钮恢复蓝色

## 主要更新

### 1. 文章支持多分类 / 多专栏

文章模型新增：

```ts
categories?: string[];
```

同时保留：

```ts
category: string;
```

兼容规则是：

- 如果只传 `category`，服务端会转成 `categories: [category]`
- 如果传 `categories`，服务端会去重、trim、过滤空字符串
- `category` 始终同步为 `categories[0]`
- 创建文章时至少需要一个分类 / 专栏

这保证旧前端、旧导出、RSS 和旧集成继续读取 `category` 时不会中断。

### 2. PostgreSQL 结构化表增加 categories 数组

`vanblog_articles` 会自动增加：

```sql
categories TEXT[] NOT NULL DEFAULT ARRAY[]::text[]
```

启动时会把旧数据回填：

```sql
categories = ARRAY[category]::text[]
```

后续写入会同时保存：

- `category = categories[0]`
- `categories = 完整分类数组`

因此当前不使用 MongoDB 的部署也可以直接使用多分类能力。

### 3. 分类查询与统计识别多分类

分类筛选和分类页查询从单纯匹配 `category` 改为兼容：

```sql
category = name OR name = ANY(categories)
```

这意味着一篇文章属于 `前端` 和 `React` 时，会同时出现在两个分类页。

分类重命名、删除前检查、分类汇总、分类归档和统计饼图也会按 `categories` 处理。

### 4. 后台文章表单支持直接创建分类 / 专栏

后台文章相关表单做了这些调整：

- “分类”文案改为“分类 / 专栏”
- 控件支持搜索已有分类
- 控件支持多选标签展示
- 控件支持直接输入新分类名
- 提交前会自动调用分类创建接口补齐不存在的分类
- 文章列表、移动端卡片、编辑页都能显示多个分类

文章筛选仍然是单分类筛选，但结果会包含任意分类命中的文章。

### 5. 前台展示、搜索与订阅同步支持 categories

前台以下位置已经优先读取 `categories`，没有该字段时回退到 `category`：

- 首页文章卡片
- 文章详情页
- 分类页与分类归档页
- 搜索结果与搜索索引
- RSS item category
- Sitemap 分类归档 URL

分类加密规则也按更严格的兼容策略处理：只要文章所属任意分类是私有分类，公开侧就按私密文章处理。

### 6. 颜色回归修复

多分类展示调整后，恢复了两处颜色表现：

- 前台文章卡片和详情页里的分类链接保持蓝色
- 后台暗色模式下主按钮默认色恢复为蓝色 `#60a5fa`

## 配置与兼容性

这一版会引入 PostgreSQL 表结构兼容迁移：

- 新增 `vanblog_articles.categories`
- 自动从旧 `category` 回填

迁移是追加字段与回填，不删除旧字段。旧接口、旧导出和旧前台仍可读取 `category`。

如果外部集成只读取 `category`，它只能看到第一个分类；要获取完整分类列表，请读取 `categories`。

## docker compose 配置单

如果你使用锁版镜像部署：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.6.2-<image-id>
```

升级后首次启动服务时会自动确保结构化表字段存在并回填旧数据。

## 升级建议

升级到 `v1.6.2` 后，建议重点验证：

1. 后台新建文章可选择多个分类 / 专栏
2. 后台可直接输入新分类名并创建文章
3. 同一篇文章能出现在多个 `/category/:name` 页面
4. 文章详情页与首页卡片显示多个分类链接
5. RSS、Sitemap、Search Index 能正常生成
6. 后台暗色模式下“新建文章”等主按钮仍是蓝色

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.6.2-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

注意：回滚旧镜像后，旧版本会继续使用 `category` 字段；已经写入的 `categories` 数组字段不会被旧版本读取为完整多分类。
