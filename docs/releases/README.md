# 发布说明

这里收录当前仓库的重要版本说明与迁移指引。

- 默认维护分支：`master`
- tag 保持独立管理，不受分支切换影响
- 如果你要做旧版迁移，请优先查看 `v1.0.3` 与 `v1.1.0`

## 关键版本

- [`v1.0.3`](./v1.0.3.md)：旧版 MongoDB 架构下的稳定导出桥接版本
- [`v1.1.0`](./v1.1.0.md)：PostgreSQL + Redis 新架构版本
- [`v1.2.1`](./v1.2.1.md)：Cloudflare 缓存优先优化与稳定归档路由版本
- [`v1.3.0`](./v1.3.0.md)：Node 24 升级、宿主机调试链路与后台兼容性修复版本
- [`v1.3.2`](./v1.3.2.md)：后台编辑器稳定性、暗色主题补齐与发布流程收敛版本
- [`v1.3.3`](./v1.3.3.md)：后台保存链路、私密文档信息修改与 PG mindmap 修复版本
- [`v1.3.4`](./v1.3.4.md)：Waline 密钥自动持久化与 latest 部署模板版本
- [`v1.3.5`](./v1.3.5.md)：PG 数据一致性修复与后台列表轻量化版本
- [`v1.3.6`](./v1.3.6.md)：站点图标/背景图生效修复与 AI 打标错误透传版本
- [`v1.3.7`](./v1.3.7.md)：Markdown 主题统一、后台独立 favicon 与代码块 / Mermaid 体验收口版本
- [`v1.3.8`](./v1.3.8.md)：暗色主题视觉重构、前台卡片背景可配置与移动端阅读体验修复版本
- [`v1.3.9`](./v1.3.9.md)：前台文章稳定性、代码块工具条轻量化与语言标签原样显示版本
- [`v1.3.10`](./v1.3.10.md)：后台主题可定制、Mermaid 明暗双主题与 Markdown 主题缓存收敛版本

## 推荐阅读顺序

1. 先看 [`v1.0.3`](./v1.0.3.md)，确认旧站如何导出 JSON
2. 再看 [`v1.1.0`](./v1.1.0.md)，确认新架构如何导入与部署
3. 再看 [`v1.2.1`](./v1.2.1.md)，确认新版本的缓存优化、归档路由与镜像部署方式
4. 再看 [`v1.3.0`](./v1.3.0.md)，确认 Node 24、宿主机调试与后台兼容修复
5. 再看 [`v1.3.2`](./v1.3.2.md)，确认后台编辑器、暗色主题与当前多镜像发布方式
6. 再看 [`v1.3.3`](./v1.3.3.md)，确认后台保存、私密文档与 PG mindmap 的最新修复
7. 再看 [`v1.3.4`](./v1.3.4.md)，确认 Waline 密钥自动生成与 latest 部署模板
8. 再看 [`v1.3.5`](./v1.3.5.md)，确认 PG 一致性修复与后台大列表优化
9. 再看 [`v1.3.6`](./v1.3.6.md)，确认 favicon / 背景图刷新链路与 AI 打标错误提示修复
10. 再看 [`v1.3.7`](./v1.3.7.md)，确认 Markdown 主题统一、后台品牌图标与代码块 / Mermaid 最新修复
11. 再看 [`v1.3.8`](./v1.3.8.md)，确认暗色主题配色、前台背景色变量与移动端性能补丁
12. 再看 [`v1.3.9`](./v1.3.9.md)，确认前台文章页稳定性修复、代码块工具条轻量化与语言标签原样显示
13. 再看 [`v1.3.10`](./v1.3.10.md)，确认后台主题可定制、Mermaid 明暗双主题与 Markdown 主题缓存策略
14. 最后结合根目录 [`RELEASE.md`](../../RELEASE.md) 与 [`DEPLOY.md`](../../DEPLOY.md) 执行发布或上线
