# v1.3.9

`v1.3.9` 是在 `v1.3.8` 基础上，围绕前台文章稳定性、代码块交互细节、后台预览一致性，以及 Markdown 目录/主题边角问题做的一次正式补丁发布。

如果只用一句话概括这个版本：

> `v1.3.9` 重点解决“某些文章在前台直接报错打不开、代码块工具条太大太重、语言标签被强制大写、后台预览与前台细节不一致”这一批已经影响实际使用体验的问题。

## 本版摘要

本次版本主要集中在四个方向：

- 修复前台文章页目录定位逻辑，避免标题内容带特殊字符时触发非法选择器，导致文章页直接报错
- 继续收紧代码块工具条视觉，把复制 / 自动换行 / 展开代码做成更轻量、更贴边的角标式交互
- 去掉代码语言标签的胶囊包裹与强制大写，按用户原始 Markdown 语言标记显示
- 统一后台预览、私密文档查看器和前台文章页的代码块表现，减少“同一篇内容前后台长得不一样”的问题

## 相比 v1.3.8 的变化

如果你当前线上或镜像分发仍以 `v1.3.8` 为基线，那么 `v1.3.9` 相比 `v1.3.8` 建议重点关注下面这些变化：

### 1. 修复前台文章页在真实内容下的致命报错

这次补丁里最重要的一项，不是样式，而是前台可用性。

在 `v1.3.8` 的一类真实文章内容里，如果标题或目录定位用到的 `data-id` 含有引号等特殊字符，前台目录组件会拼出非法的 `querySelectorAll` 选择器，进而让整篇文章页直接报：

- `This page couldn’t load`
- 或前台进入文章页后白屏 / Internal Server Error

`v1.3.9` 对目录定位逻辑做了安全转义处理，并补了回归测试，重点覆盖：

- 标题包含引号、路径片段、命令片段时的目录同步
- 前台文章页目录高亮与滚动定位
- 线上内容不再因为单个标题文本而整页不可访问

如果你线上最近遇到“只有某一篇文章打不开，而别的文章正常”的情况，这版就是直接针对这类问题的修复。

### 2. 代码块工具条从“能用”继续收口到“轻且不打扰”

`v1.3.8` 已经把代码块的复制、自动换行、展开/收起改成图标按钮，但真实试用后又暴露出几个问题：

- 右上角一排工具仍然偏大，抢正文视觉
- 语言类型（如 `bash`）被做成了明显的胶囊块，观感偏重
- 某些地方会把 `bash` 强制显示成 `BASH`，不符合原始输入
- 后台预览区和前台虽然都支持图标按钮，但尺寸与细节不够统一

`v1.3.9` 继续把这套工具条收紧：

- 按钮整体进一步缩小，贴近代码块右上角
- 去掉外层大胶囊/大圆角包裹感
- 语言标签不再使用胶囊背景，只保留轻量文字角标
- 语言大小写保留用户原始写法，不再强制转成大写

整体效果会更接近“辅助操作控件”，而不是悬浮在代码块上的一排主角组件。

### 3. 后台预览与文档查看器不再出现代码块交互退化

这一轮还顺手修了后台预览链路里的几个一致性问题：

- 某些预览场景里，“展开代码”按钮可能退回成文字，而不是图标
- 私密文档 / 文档查看器中的代码块尺寸与编辑器预览不一致
- 代码块的角标位置、图标尺寸、折叠按钮显示方式前后台不统一

`v1.3.9` 把后台编辑器预览、私密文档预览和前台展示重新对齐到同一套细化后的代码块样式，让预览更接近最终前台效果。

### 4. Markdown 主题 hotfix 和暗色细节继续做减法

`v1.3.8` 已经做过一轮 Markdown 主题统一，但在真实内容和暗色模式下，仍然会有一些“边角显得太重”的问题。

这一版补丁的思路不是再加更多装饰，而是继续做减法：

- 降低代码块工具条存在感
- 让语言标签回到“说明信息”而不是“装饰块”
- 保持暗色代码块按钮 hover / 激活态，但压低体积和视觉噪声

这使得前台文章页在深色主题下更耐看，正文不会被控件打断。

## 主要特性 / 修复

### 前台稳定性

- 修复 `MarkdownTocBar` 在标题含特殊字符时生成非法 DOM 选择器的问题
- 补充目录定位安全转义与回归测试，避免线上个别文章整页不可访问

### 代码块 / 阅读体验

- 继续缩小代码块右上角工具条尺寸，减少遮挡感
- 语言标签去掉胶囊包裹，改成轻量角标文字
- 语言标签不再强制转大写，显示原始语言字符串
- 前台、后台预览、文档查看器统一更新代码块按钮尺寸、间距与定位
- 修复后台预览区折叠按钮在部分场景退回文字按钮的问题

### 后台预览 / Markdown 呈现

- 后台编辑器预览、文档查看器与前台的代码块交互进一步统一
- 主题 hotfix 继续收口，减少暗色模式下控件层次过重的问题

## 部署前注意事项

发布 `v1.3.9` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.9`
- Git tag 使用 `v1.3.9`
- Docker Hub 推送仍以本机 `pnpm release:images:push` 为准
- 当前正式发布仍默认只推 `linux/amd64`

如果你的站点最近遇到下面任意现象，这版建议优先升级：

- 前台访问某篇文章时出现 `This page couldn’t load`
- 代码块右上角按钮太大、太显眼，影响正文阅读
- `bash` / `text` 这类语言标签在前台或预览区被强制显示成大写
- 后台预览与前台文章页的代码块交互不一致

- `docker-compose.image.yml`

## 升级步骤

如果你当前运行的是 `v1.3.8`：

1. 备份当前 `data/`、`caddy/`、`log/` 目录
2. 将发布镜像标签切到 `v1.3.9`
3. 执行：

```bash
docker compose -f docker-compose.image.yml pull
docker compose -f docker-compose.image.yml up -d
```

如果你使用不可变 tag：

```bash
export VANBLOG_DOCKER_REPO=kevinchina/deeplearning
export VANBLOG_RELEASE_SUFFIX=v1.3.9-<image-id>
docker compose -f docker-compose.image.yml up -d
```

## 回滚方法

如果升级后需要回滚，直接把 `VANBLOG_RELEASE_SUFFIX` 改回上一版即可：

```bash
export VANBLOG_DOCKER_REPO=kevinchina/deeplearning
export VANBLOG_RELEASE_SUFFIX=v1.3.8-<image-id>
docker compose -f docker-compose.image.yml up -d
```

如果你使用的是版本别名，也可以临时改回：

```bash
export VANBLOG_RELEASE_SUFFIX=v1.3.8
docker compose -f docker-compose.image.yml up -d
```
