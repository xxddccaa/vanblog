# v1.3.10

`v1.3.10` 是在 `v1.3.9` 基础上，围绕后台主题定制、Mermaid 明暗双主题渲染、Markdown 主题缓存失效，以及前台卡片背景色联动做的一次正式补丁发布。

如果只用一句话概括这个版本：

> `v1.3.10` 重点解决“后台和前台主题能力分散、Mermaid 在亮色/暗色博客里容易失真、Markdown 主题 CSS 容易被缓存、前台卡片暗色背景不够统一”这一批实际部署和日常使用中的体验问题。

## 本版摘要

本次版本主要集中在四个方向：

- 新增后台独立主题配置，支持分别定制亮色/暗色主色与背景色，并把登录页、初始化页、文档页和运行中的后台界面统一到同一套色板
- Mermaid 改为跟随当前主题切换 Mermaid 官方 light/dark 配置，前台文章页、后台编辑器预览和文档查看器保持一致
- 所有 `/markdown-themes/` 主题资源统一追加稳定版本参数，降低 Cloudflare 或浏览器缓存导致旧 CSS 继续生效的问题
- 前台卡片背景色在深色模式下继续收口到同一套蓝青系 surface 变量，减少“文章卡片改了，但外围壳层还是另一套底色”的割裂感

## 相比 v1.3.9 的变化

如果你当前线上或镜像分发仍以 `v1.3.9` 为基线，那么 `v1.3.10` 相比 `v1.3.9` 建议重点关注下面这些变化：

### 1. 后台终于有了独立且可持久化的主题定制能力

这一版最直观的变化，是后台不再只能跟着预设主题走，而是可以单独配置后台自己的品牌色与背景色。

`v1.3.10` 新增了后台主题设置链路，覆盖：

- 后台亮色主色
- 后台暗色主色
- 后台亮色背景色
- 后台暗色背景色
- 恢复默认主题色

这些设置会持久化到服务端，并通过公开站点信息接口下发，让后台登录页、初始化页、文档页、编辑器、侧边栏和常用操作面板都使用同一套颜色变量，而不是各自写死一份颜色。

如果你需要把后台管理界面更接近自己的品牌色，或者希望暗色后台不再沿用默认近黑背景，这一版就是直接针对这类诉求的补丁。

### 2. Mermaid 改成官方明暗双主题，不再靠手工覆写 SVG 颜色硬撑

过去这条链路里，Mermaid 更像是在一个固定浅色底板上“强行兼容”前台和后台：

- 亮色站点还能看，但一旦切到暗色主题，图表文字、边框和背景容易出现违和
- 前台文章页、后台编辑器预览、文档查看器的 Mermaid 呈现不完全一致
- 为了救局部场景，样式里出现了较多直接改 SVG 文本/描边颜色的覆写

`v1.3.10` 的做法是换思路：

- 跟随当前主题模式切换 Mermaid 官方 `default` / `dark` 配置
- 在主题切换时重新绑定 Mermaid 插件，保证重新渲染后的 SVG 使用正确配色
- Mermaid 容器 CSS 只负责留白、边框和布局，不再大面积强改图内元素颜色

这样处理后，亮色和暗色博客都能得到更接近 Mermaid 官方预期的图表效果，而且后台预览与前台展示更容易保持一致。

### 3. Markdown 主题 CSS 增加版本参数，减少“明明发版了但用户还在看旧主题”的缓存问题

这次补丁还顺手处理了一个很常见但很烦的发布问题：

- `/markdown-themes/*.css` 这类托管在站点内的主题资源，经过 Cloudflare 或浏览器缓存后，发版后可能继续命中旧文件
- 前台和后台都可能出现“代码已经更新，但主题样式没刷新”的错觉

`v1.3.10` 统一通过版本化 helper 给受管的 Markdown 主题资源追加稳定查询参数，只对 `/markdown-themes/` 下的资源生效，不影响外部 CDN 地址。

对正式发版来说，这意味着：

- 主题 CSS 能更明确地跟随当前版本刷新
- 后台预览、前台文章页和发布后的真实访客更容易看到同一套主题效果
- 以后维护 `/markdown-themes/` 下的 hotfix CSS 时，缓存可控性更高

### 4. 前台卡片背景色继续向“整套 surface 联动”收口

前面的版本已经支持前台卡片背景色配置，但在深色模式下，常见问题还是会出现在“卡片改了，周边壳层和 hover/border 还是另一套默认颜色”。

`v1.3.10` 继续把这块收口：

- 基于前台亮色/暗色卡片背景色推导更多 surface 变量
- 让文章卡片、导航壳层、阴影、边框、hover 与页面背景更接近同一色系
- 暗色模式继续保持蓝青系阅读氛围，而不是回退到近黑外壳

如果你已经在用自定义前台卡片背景，这版会让整体观感更统一；如果你正准备启用这项配置，也更适合作为新默认基线。

## 主要特性 / 修复

### 后台主题与管理界面

- 新增后台主题设置读写与重置接口，支持服务端持久化后台亮/暗主色与背景色
- 后台登录页、初始化页、文档页、编辑器和全局布局改为跟随后台主题变量
- 后台主题配置通过公开站点信息接口下发，前后端初始化链路保持一致
- 新增颜色输入组件与颜色合法性校验，减少手输配置时的格式错误

### Mermaid / Markdown 呈现

- 前台与后台 Mermaid 改用 Mermaid 官方 light/dark 主题映射
- 主题切换时重新绑定 Mermaid 插件，避免亮暗切换后继续沿用旧 SVG 配色
- Mermaid 容器样式收口为留白、边框、滚动和布局控制，不再大面积覆写图内文字/线条颜色
- Markdown 主题资源统一追加版本参数，降低缓存导致的旧样式残留
- 补充 Mermaid 主题切换、Markdown 主题版本化与后台设置链路相关测试

### 前台阅读体验

- 前台卡片亮/暗背景色继续联动更多 surface 变量，减少壳层和正文色系割裂
- 布局头部主题资源同步逻辑继续收紧，确保切换主题时样式链接与 hotfix 链接一致更新

## 部署前注意事项

发布 `v1.3.10` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.10`
- Git tag 使用 `v1.3.10`
- Docker Hub 推送仍以本机 `pnpm release:images:push` 为准
- 当前正式发布仍默认只推 `linux/amd64`

如果你的站点最近遇到下面任意现象，这版建议优先升级：

- 博客同时支持亮色/暗色，但 Mermaid 在不同主题下颜色或可读性不稳定
- 后台登录页、初始化页、编辑器和正文页想统一品牌色，但目前只能靠分散样式修改
- Cloudflare 或浏览器缓存导致 Markdown 主题 CSS 发布后不立即刷新
- 深色前台自定义卡片背景后，外围壳层、hover 或边框仍然像另一套默认主题

## 升级步骤

如果你当前运行的是 `v1.3.9`：

1. 备份当前 `data/`、`caddy/`、`log/` 目录
2. 将发布镜像标签切到 `v1.3.10`
3. 执行：

```bash
docker compose -f docker-compose.image.yml pull
docker compose -f docker-compose.image.yml up -d
```

如果你使用不可变 tag：

```bash
export VANBLOG_DOCKER_REPO=kevinchina/deeplearning
export VANBLOG_RELEASE_SUFFIX=v1.3.10-<image-id>
docker compose -f docker-compose.image.yml up -d
```

## 回滚方法

如果升级后需要回滚，直接把 `VANBLOG_RELEASE_SUFFIX` 改回上一版即可：

```bash
export VANBLOG_DOCKER_REPO=kevinchina/deeplearning
export VANBLOG_RELEASE_SUFFIX=v1.3.9-<image-id>
docker compose -f docker-compose.image.yml up -d
```

如果你使用的是版本别名，也可以临时改回：

```bash
export VANBLOG_RELEASE_SUFFIX=v1.3.9
docker compose -f docker-compose.image.yml up -d
```
