# v1.4.3

`v1.4.3` 是一次偏后台体验的补丁发布，重点修掉管理员登录态过期后 `/admin/article` 等管理页容易表现成“列表空了、但不知道其实已经掉线”的问题。

如果只用一句话概括这个版本：

> `v1.4.3` 让后台登录过期时可以被明确感知：自动清理旧 token、直接跳回登录页、提示“登录已过期”，并在重新登录后回到原来的管理页面。

## 本版摘要

这次补丁主要集中在三个方向：

- 后台请求层统一识别登录态失效，不再让文章列表之类页面静默落成空状态
- 登录页增加一次性过期提示，并保留原始后台地址用于重新登录后的回跳
- 补齐针对登录过期跳转链路的自动化测试，避免以后又回到“只有手动登出再登录才恢复”的体验

## 相比 v1.4.2 的变化

如果你已经在使用 `v1.4.2` 的 AI 工作台和浏览器终端，这一版不新增部署拓扑，也不改镜像数量；它主要是把后台已有登录会话的失效处理补完整。

### 1. 管理页不再把登录过期误表现成空列表

之前有一个典型场景：

- 管理员已经登录
- 本地 token 过期或被服务端作废
- 再打开 `/admin/article`、`/admin/draft` 之类页面时，请求实际上已经返回 `401 Unauthorized`
- 但前端没有把它明确收口成“需要重新登录”，页面就可能只剩空数据或异常刷新体验

`v1.4.3` 现在会在请求层统一识别这类标准鉴权失败，并立即进入重登流程。

### 2. 登录过期时会直接跳到登录页，并保留回跳地址

这一版新增了一套前端侧的统一会话失效处理：

- 收到后台 guard 返回的 `401 Unauthorized`
- 清理本地旧 token
- 记录一次性提示文案
- 整页跳到 `/admin/user/login`
- 把原始后台地址写到 `redirect` 参数

因此管理员重新登录之后，可以直接回到原来的页面，例如：

- `/admin/article`
- `/admin/draft`
- `/admin/site/setting`
- `/admin/ai`

而不需要再手工找回之前的操作入口。

### 3. 登录页会明确提示“登录已过期，请重新登录”

过去如果登录态过期，用户常常只能凭经验判断“是不是掉线了”。这版把提示也补上了：

- 登录页会显示一次性的 `登录已过期，请重新登录`
- 提示只消费一次，不会在后续正常打开登录页时反复出现
- 普通密码错误、权限不足、演示站限制等其他错误，仍然保持原来的提示逻辑，不会误报成登录过期

## 主要特性 / 修复

### 后台登录态处理

- 请求层统一识别标准 `401 Unauthorized`
- 初始化阶段如果发现后台会话已失效，会直接走重登流程
- 不再依赖“先手动登出、再重新登录”才能恢复文章列表

### 登录页体验

- 登录页新增过期提示
- 登录成功后继续复用 `redirect` 回跳
- 后台子路径仍保持 `/admin/...` 语义，不破坏现有 Caddy 子路径部署

### 自动化测试

- 新增针对登录过期识别、提示存取、登录页跳转地址的测试
- 继续保留现有 TypeScript 检查与部署文档断言

## 部署与兼容性说明

`v1.4.3` 没有改变下面这些发布 / 部署边界：

- 默认公开拓扑仍然只有 `caddy`、`server`、`website`、`admin`、`waline`、`postgres`、`redis`
- AI 工作台仍然是按需叠加 `docker-compose.ai-qa.yml` / `docker-compose.fastgpt.yml`
- bundled FastGPT 仍然不是默认公开拓扑的一部分
- `pnpm release:images` / `pnpm release:publish` 仍然只发布 VanBlog 核心 5 个镜像

这意味着：

- 不需要修改现有 compose 编排
- 不需要新增环境变量
- 已经在使用 `v1.4.2` 的部署可以直接升级到 `v1.4.3`

## docker compose 配置单

推荐正式部署仍然使用：

- `docker-compose.image.yml`
- `.env.release.example`

核心镜像示例保持：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.4.3-<image-id>
```

主栈启动：

```bash
docker compose -f docker-compose.image.yml up -d
```

只连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验，也可以继续使用：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级步骤

1. 拉取 `v1.4.3` 对应代码或镜像
2. 确认 `.env` 中的 `VANBLOG_DOCKER_REPO` 与 `VANBLOG_RELEASE_SUFFIX`
3. 执行 `pnpm test:deploy`
4. 执行 `docker compose -f docker-compose.image.yml pull`
5. 执行 `docker compose -f docker-compose.image.yml up -d`
6. 打开 `/admin/article` 或其他管理页做一次人工验收
7. 可选：清掉旧浏览器缓存或旧后台标签页，确保拿到最新前端资源
8. 手工验证：让登录态过期后再访问后台，确认会自动跳到登录页并显示过期提示

## 回滚方法

如果线上有问题，把 `VANBLOG_RELEASE_SUFFIX` 改回上一版即可，例如：

```env
VANBLOG_RELEASE_SUFFIX=v1.4.2-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只是后台登录体验链路需要临时回退，也不需要拆掉 AI overlay；它与 `/admin/ai` 的浏览器终端能力彼此独立。
