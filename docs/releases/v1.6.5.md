# v1.6.5

`v1.6.5` 是一次前台背景图越界回弹体验修复版本，重点把背景图从 `html` canvas 上移走，只保留在 `body` 页面内容层。

如果只用一句话概括：

> `v1.6.5` 让浏览器 rubber-band 越界回弹时的空白区域只显示纯色页面底色，不再露出整张自定义背景图。

## 本版摘要

这一版主要集中在三件事：

- 前台 `html` 只保留纯色 `background-color` 和滚动边界，不再挂载背景图
- 前台背景图继续挂在 `body`，亮色 / 暗色主题下仍正常显示用户配置的 wallpaper
- 增加样式回归测试，确保背景图不会重新回到 `html` canvas 上

## 相比 v1.6.4 的变化

如果你已经在使用 `v1.6.4`，这一版最直接的变化是：

- 设置前台背景图后，触控板或部分浏览器的顶部 / 底部弹性回弹空白不再显示整张背景图
- `html` 元素作为浏览器 canvas 只绘制纯色页面底色
- `body` 仍然绘制 `--bg-image` / `--bg-image-dark`，正常页面区域的背景图不受影响
- 移动端背景图仍保留 `background-attachment: scroll` 回退

## 主要更新

### 1. 背景图只挂在 body

浏览器发生 rubber-band 越界滚动时，上下边界外的空白由 canvas 背景绘制。`html` 元素带有背景图时，这块空白会直接露出整张背景图，视觉上像页面内容被拉走。

本版让 `html` 只保留纯色背景：

```css
html {
  background-color: var(--vb-front-page-bg-light, #f4f8fb);
  overscroll-behavior-y: none;
}

html.dark {
  background-color: var(--vb-front-page-bg-dark, #0c1928);
}
```

背景图继续由 `body` 承载：

```css
body {
  background-image: var(--bg-image);
}

body.dark {
  background-image: var(--bg-image-dark);
}
```

由于 `html` 已有非透明背景色，`body` 背景不会再传播到 canvas，越界空白会显示纯色页面底色。

### 2. 增加回归测试

新增前台样式测试，覆盖：

- `html` / `html.dark` 不允许出现 `background-image`
- `body` 必须继续保留亮色背景图变量
- `body.dark` 必须继续保留暗色背景图变量
- 移动端背景附件回退规则仍然存在

## 配置与兼容性

这一版不引入数据库迁移，不改变 API、环境变量或 compose 拓扑。

如果你使用自定义 CSS 把背景图重新挂到 `html` 上，浏览器越界回弹时仍可能再次露出整张背景图。建议自定义背景图继续放在 `body` 或页面内容层。

## docker compose 配置单

如果你使用锁版镜像部署：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.6.5-<image-id>
```

## 升级建议

升级到 `v1.6.5` 后，建议重点验证：

1. 前台已配置背景图时，首页顶部连续向上滚动不会露出整张背景图
2. 前台已配置背景图时，页面底部继续向下滚动不会露出整张背景图
3. 正常页面区域仍显示用户配置的亮色背景图
4. 暗色模式下正常页面区域仍显示用户配置的暗色背景图
5. 移动端背景图滚动表现仍然流畅

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.6.5-<image-id>
```

改回上一版的 release suffix，然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```
