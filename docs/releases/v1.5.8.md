# v1.5.8

`v1.5.8` 是一次围绕默认 Markdown 主题观感的小补丁发布，重点是把 `Vanblog Default（炭黑石墨）` 和亮色默认 `Phycat Sky` 的正文排版与引用块样式收紧成更接近 Typora 的阅读风格。

如果只用一句话概括：

> `v1.5.8` 把前台/后台预览的默认暗色与亮色主题的行距、字距、引用块样式改成更接近 Typora 的观感，不改主题选择项，也不影响其它 Phycat 变体。

## 本版摘要

这一版主要集中在两件事：

- 收紧默认主题的正文排版：`line-height`、`letter-spacing`、`word-spacing` 改成更接近 Typora 默认值
- 重做默认主题的引用块：去掉 `✨` 装饰 emoji、缩小圆角与内边距、取消 hover 缩放动画，统一走 hotfix 的"左侧竖线 + 浅底色"样式

## 相比 v1.5.7 的变化

如果你已经在使用 `v1.5.7`，这一版最直接的变化会出现在前台文章页和后台编辑器的 Markdown 预览区：

- 暗色默认主题（炭黑石墨）和亮色默认主题（Phycat Sky）中的正文行距从 `2.25` 收紧到 `1.75`，`letter-spacing` 从 `1.1px` 改为 `0`，`word-spacing` 也恢复默认
- 引用块不再显示 `✨` emoji，左内边距从 `52px` 回到 `1em`，圆角从 `16px` 改为 `3px`，hover 时不再有 `scale(1.02)` 弹性动画
- 标题（`h1 ~ h6`）的视觉装饰与 hover 动画保持不变，Phycat 的视觉识别仍然保留
- 其他 Phycat 系列（Cherry、Mauve、Mint、Prussian、Forest、Sakura、Caramel 等）均未改动，`✨` 引用块、宽松行距保持原样

这不是一次新功能版本，也不是 bug 修复，而是一次针对默认主题阅读体验的排版收紧。

## 主要更新

### 1. 默认主题正文收紧成 Typora 风

修改集中在两个叶子文件，不动任何共享父样式：

- `packages/website/public/markdown-themes/vanblog-graphite-dark-only.css`
- `packages/website/public/markdown-themes/phycat-sky-light-only.css`

在文件末尾追加覆盖块，只动正文、列表和引用块的排版属性：

- `.markdown-body` 的 `line-height: 1.75`、`letter-spacing: 0`
- `.markdown-body p` 的 `word-spacing: normal`、`margin: 1em 0`
- `.markdown-body li` 的 `line-height: 1.7`

选择器特异性与父文件一致，由 CSS 源码顺序决定胜出，避免和现有颜色变量、标题装饰冲突。

### 2. 默认主题引用块 Typora 化

同样集中在上述两个叶子文件，覆盖引用块的布局与伪元素：

- `blockquote { padding: 0.6em 1em; border-radius: 3px; line-height: 1.7; }`
- `blockquote::before { content: none; }` 移除 `✨` 装饰
- `blockquote:hover { transform: none; }` 取消弹性缩放
- 保留 `vanblog-theme-hotfix.css` 统一画出的"左侧竖线 + 浅底色"，它用 `!important` 覆盖所有主题的引用块边框与背景，不受本次改动影响

### 3. 保持其它 Phycat 变体与主题选择项稳定

- 没有改 `phycat-dark-only.css` / `phycat-cherry-light-only.css` 这两个共享基类
- 没有改 `vanblog-theme-hotfix.css`
- 没有动 `MARKDOWN_THEME_DEFAULTS`、admin `MarkdownThemeSelector` 下拉项或主题注册
- `packages/website/__tests__/markdown-theme.spec.ts` 中对 `vanblog-graphite-dark-only` 的 id 断言继续通过

## 配置与兼容性

`v1.5.8` 不引入新的环境变量，也不需要数据库迁移：

- 不新增 `.env` 配置项
- 不需要数据库结构迁移
- 不改变 AI / FastGPT 部署方式
- 从 `v1.5.7` 可直接升级

缓存：CSS 资源通过 `?vbv=` 查询参数带版本号，`NEXT_PUBLIC_MARKDOWN_THEME_ASSET_VERSION` 默认取根 `package.json#version`。升级到 `v1.5.8` 后 URL 会变成 `?vbv=1.5.8`，Cloudflare 与浏览器缓存都会自然 bust，无需手工清理。

## 发布与部署边界

`v1.5.8` 继续保持当前多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 继续使用官方镜像
- AI 工作台继续随 `server` / `admin` 核心镜像发布
- bundled FastGPT 仍然是可选叠加，不属于默认主栈公开拓扑

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.5.8-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级建议

升级到 `v1.5.8` 后，建议重点看这几个地方：

1. 前台文章正文（中英混排、列表、表格）在暗色和亮色模式下是否更接近 Typora 的行距/字距观感
2. `>` 引用块不再带 `✨`，变成左侧竖线 + 浅底色，hover 时不再缩放
3. 标题（`h1 ~ h6`）的装饰条、圆点、hover 动画继续保留
4. 切到 `Vanblog Plain（朴素/经典）` 或其它 Phycat 变体时，观感保持原样，仅默认主题发生变化

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.7-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果前台观感变化仅仅是因为浏览器缓存了旧的 `?vbv=1.5.7` CSS，升级后新 URL 参数会自动 bust 旧缓存，通常不需要手工清理。
