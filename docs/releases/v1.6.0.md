# v1.6.0

`v1.6.0` 是一次部署边界与后台能力收口版本：它把 `/admin/ai` 这套 AI 工作台从当前主线彻底移除，同时保留原有的 `ai-tagging` 能力，并把多容器与 all-in-one 的部署文档、测试和运行时重新统一到同一套无 AI 口径。

如果只用一句话概括：

> `v1.6.0` 删除了 `/admin/ai`、FastGPT overlay、浏览器终端和相关文档，但不影响博客主栈，也不影响 `ai-tagging` 页面与接口继续使用。

## 本版摘要

这一版主要集中在三件事：

- 删除后台 `/admin/ai` 页面、后端 AI-QA 接口和相关同步任务
- 删除 FastGPT / ai-terminal 的运行时入口、compose 文件、Caddy 转发和部署文档
- 保留 `ai-tagging` 现有功能，并确认多容器和 all-in-one 部署路径仍然可发布、可测试

## 相比 v1.5.8 的变化

如果你已经在使用 `v1.5.8`，这一版最直接的变化会出现在后台与部署层：

- `/admin/ai` 不再作为当前版本的后台能力存在
- `docker-compose.ai-qa.yml`、`docker-compose.fastgpt.yml`、`docker-compose.latest.ai.yml` 已从仓库删除
- `docker/server/terminal-shell.sh`、`docker/fastgpt/*`、Caddy `/admin/ai-terminal` 转发规则已删除
- `README.md`、`DEPLOY.md`、`RELEASE.md` 以及当前 guide / faq / reference 文档已全部改成无 AI 版本
- `ai-tagging` 保持原样，文章打标相关页面、接口和 provider 没有删除

这不是一次样式调优版本，而是一次明确收缩产品边界、简化部署面和维护面的版本。

## 主要更新

### 1. 删除 `/admin/ai` 工作台

移除内容包括：

- `packages/admin/src/pages/AIQA/*`
- `packages/admin/src/services/van-blog/ai-qa.js`
- `packages/server/src/controller/admin/ai-qa/*`
- `packages/server/src/provider/ai-qa/*`
- `packages/server/src/schedule/ai-qa.task.ts`
- `packages/server/src/types/ai-qa.dto.ts`

同时清理了：

- 后台菜单与路由中的 `/ai`
- 默认菜单配置中的 `ai` 项
- 文章、草稿、私密文档在增删改时触发的 AI 知识同步调用

### 2. 删除 AI 运行时与部署残留

仓库中已移除：

- `docker-compose.ai-qa.yml`
- `docker-compose.fastgpt.yml`
- `docker-compose.latest.ai.yml`
- `docker/server/terminal-shell.sh`
- `docker/fastgpt/bootstrap-team-free-plan.js`
- `docker/fastgpt/config.json`
- `docker/fastgpt/config.json.example`

同时更新了：

- `docker/server.Dockerfile`：移除 `wetty`、`opencode-ai` 和 `7681` 暴露
- `docker/server/entrypoint.sh`：只启动主 server 进程
- `docker/caddy/Caddyfile` 与 `docker/caddy/Caddyfile.https`：移除 `/admin/ai-terminal` 路由与 `forward_auth`
- `docker/all-in-one.Dockerfile` 与 `docker/all-in-one/entrypoint.sh`：移除 AI terminal 相关入口

### 3. 保留 `ai-tagging`

这一版没有删除也没有弱化 `ai-tagging`：

- `packages/admin/src/pages/DataManage/tabs/AITagging.jsx`
- `packages/admin/src/services/van-blog/ai-tagging.js`
- `packages/server/src/controller/admin/ai-tagging/*`
- `packages/server/src/provider/ai-tagging/*`

换句话说，本次删除范围仅限 `/admin/ai` 这套工作台，不包括文章 AI 打标。

### 4. 部署与测试口径统一

`tests/deployment-config.test.mjs` 已重写为当前现实：

- 断言 AI 专属 compose / docs / terminal 文件已经不存在
- 断言 Caddy 不再暴露 `/admin/ai-terminal`
- 断言 server runtime 不再包含 `wetty` / `opencode-ai`
- 断言当前生效文档不再提及 `/admin/ai` / FastGPT overlay

all-in-one 路径也同步验证：

- 仍然是可用发布物
- 仍然使用 `vanblog-all-in-one-*` 标签
- 不再包含 AI 终端或 AI overlay 相关约束

## 配置与兼容性

`v1.6.0` 不引入新的环境变量，也不需要数据库迁移：

- 不新增 `.env` 配置项
- 不需要数据库结构迁移
- 旧的 AI 相关数据库数据会继续保留，但当前代码不再读取或维护它们
- 从 `v1.5.8` 可直接升级

如果你的环境里以前手工保留过 `/admin/ai` 相关旧前端缓存，升级后建议做一次浏览器强刷。

## 发布与部署边界

`v1.6.0` 继续保持当前多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 继续使用官方镜像
- `vanblog-all-in-one` 仍然是独立可选发布物
- 当前版本不再提供 `/admin/ai`、FastGPT overlay、浏览器终端

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.6.0-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

all-in-one 部署：

```bash
docker compose -f docker-compose.all-in-one.image.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.yml up -d
```

all-in-one latest：

```bash
docker compose -f docker-compose.all-in-one.latest.yml up -d
```

## 升级建议

升级到 `v1.6.0` 后，建议重点看这几个地方：

1. 后台左侧菜单里不再出现 AI 工作台入口
2. 直接访问 `/admin/ai` 时，不应再出现旧的 AI 控制台
3. `ai-tagging` 页面与接口仍然可正常使用
4. `docker compose` 多容器路径和 `all-in-one` 单容器路径都能正常启动
5. `server` 容器不再暴露 `7681` 终端端口

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.8-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只是浏览器缓存还残留旧后台资源，优先强刷浏览器缓存，不要误判成部署未生效。
