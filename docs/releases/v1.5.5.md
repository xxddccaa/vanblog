# v1.5.5

`v1.5.5` 是一次聚焦后台文章编辑体验的补丁发布，核心目标只有一个：把编辑区和预览区的同步滚动恢复到可用状态。

如果只用一句话概括：

> `v1.5.5` 修掉了文章编辑页里 ByteMD 预览区的双滚动容器问题，让编辑区和预览区重新按同一滚动层联动。

## 本版摘要

这一版主要集中在三件事：

- 修复文章编辑页里编辑区和预览区不同步滚动的问题
- 收敛 ByteMD 预览链路中的双滚动容器，避免外层同步、内层实际滚动的错位
- 为 `DocumentViewer` 增加显式布局模式，区分“自带滚动容器”和“继承父级滚动容器”两种使用方式

## 相比 v1.5.4 的变化

如果你已经在使用 `v1.5.4`，这一版最直接的变化只会出现在后台文章编辑页：

- 滚动左侧编辑区时，右侧预览区会重新按比例同步
- 滚动右侧预览区时，不再因为内部还有一层独立滚动容器而和编辑区脱节
- `DocumentViewer` 在编辑器预览场景下会继承父级预览面板滚动，而不是再额外创建第二层滚动

这不是一次功能扩展版本，而是一次针对编辑体验退化的定点修复发布。

## 主要更新

### 1. 修复 ByteMD 预览双滚动层

这次问题的根因不是“同步逻辑完全消失”，而是同步逻辑和真实滚动层不一致：

- ByteMD 自己的预览面板已经有一层滚动容器
- 预览面板内部又嵌了一层会自己滚动的 `DocumentViewer`
- 结果同步逻辑控制的是外层，用户实际滚动的是内层，所以联动看起来像是失效了

`v1.5.5` 的处理方式是：

- 保留 `DocumentViewer` 的 Markdown 渲染能力
- 但在编辑器预览场景里让它继承父级滚动容器
- 不再在 ByteMD 预览区里制造第二层纵向滚动

这样编辑器同步滚动会重新落回同一套滚动坐标系。

### 2. `DocumentViewer` 增加滚动容器布局模式

这一版给 `DocumentViewer` 增加了显式布局分支：

- 默认模式仍然保持“组件自己负责滚动”
- 编辑器预览场景改成“继承父级滚动”

这样不会影响文档页、AI 富文本展示等原有用法，也避免后续别的场景再次因为容器嵌套把滚动行为带偏。

### 3. 补充布局约束测试

为了避免以后再次把预览区改回双滚动结构，这一版补了一个针对 `DocumentViewer` 布局模式的小测试，专门约束：

- 默认场景使用自身滚动容器
- 编辑器预览场景使用父级滚动容器

这类测试不大，但对回归问题很有价值。

## 配置与兼容性

`v1.5.5` 不引入新的环境变量，也不需要数据库迁移：

- 不新增 `.env` 配置项
- 不需要数据库结构迁移
- 不改变 AI / FastGPT 部署方式
- 从 `v1.5.4` 可直接升级

需要注意的一点是：

- 这一版主要修的是后台编辑页交互问题；如果你线上已经在 `v1.5.4`，升级后重点验证编辑器滚动联动即可

## 发布与部署边界

`v1.5.5` 继续保持当前多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 继续使用官方镜像
- AI 工作台继续随 `server` / `admin` 核心镜像发布
- bundled FastGPT 仍然是可选叠加，不属于默认主栈公开拓扑

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.5.5-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级建议

升级到 `v1.5.5` 后，建议重点看这几个地方：

1. 文章编辑页中，滚动左侧编辑区时右侧预览区是否重新联动
2. 滚动右侧预览区时，左侧编辑区是否保持同步，而不是出现一边滚一边不动
3. 代码块、Mermaid、数学公式和图片在预览区里是否仍然正常渲染
4. 文档页、AI 富文本展示等仍然使用 `DocumentViewer` 的页面是否没有被这次滚动改动带出回归

如果浏览器仍显示旧行为，优先强刷管理端静态资源缓存后再确认。

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.4-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只有后台浏览器仍表现异常，优先排查浏览器缓存或旧静态资源，而不是先判断镜像回滚失败。
