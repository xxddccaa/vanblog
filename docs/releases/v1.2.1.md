# v1.2.1

`v1.2.1` 是围绕“Cloudflare 缓存优先”和“镜像部署可追踪”继续推进的稳定版本。

这个版本的重点不是继续堆实时 SSR，而是尽量把公开页面变成更稳定的 HTML，把高频变化模块拆成独立碎片接口，让 Cloudflare 更容易直接命中缓存。

## 核心变化

- 文章页继续保持服务端输出正文 HTML，但把高频变化块进一步拆成独立请求
- 新增文章碎片接口，异步加载：
  - 上一篇 / 下一篇
  - 阅读量 / 访客量 / 评论数
  - 相关文章 / 最新文章 / 热门文章
- 公开接口统一补齐面向 CDN 的缓存头、ETag、Last-Modified、Cache-Tag
- 新增 Cloudflare cache purge provider，可在文章和静态产物变更后做更精细的缓存失效
- 首页和公共列表页继续朝“稳定 shell + 异步碎片”方向收敛，减少因局部变化导致的整页失效
- 新增稳定归档路由：
  - `/archive`
  - `/archive/<year>`
  - `/archive/<year>/<month>`
  - `/category/<category>/archive/<year>/<month>`
  - `/tag/<tag>/archive/<year>/<month>`
- 旧的深分页路由不再作为主入口，改为重定向到归档路径
- 前台 `/moment` 页面文案统一为“动态”，后台管理仍保持“个人动态”

## 为什么推荐升级到这版

如果你的站点链路是：

`用户 -> Cloudflare -> Caddy -> 内网服务`

那么 `v1.2.1` 相比之前的版本，更适合下面这种目标：

- 文章正文页长期稳定，可强缓存
- 阅读量、评论数、推荐文章等动态信息不再拖累整页 HTML
- 发布新文章时，尽量少让旧页面缓存失效
- 首页只承担“最新内容入口”，历史内容迁移到更稳定的归档页
- 后续可以更自然地配合 Cloudflare Cache Rules / Cache-Tag / Purge 使用

## 推荐镜像标签

本次版本推荐优先使用不可变标签：

```text
v1.2.1-<image-id>
```

例如：

```text
kevinchina/deeplearning:vanblog-caddy-v1.2.1-<image-id>
kevinchina/deeplearning:vanblog-server-v1.2.1-<image-id>
kevinchina/deeplearning:vanblog-website-v1.2.1-<image-id>
kevinchina/deeplearning:vanblog-admin-v1.2.1-<image-id>
kevinchina/deeplearning:vanblog-waline-v1.2.1-<image-id>
```

如果你已经知道本次发布的具体 suffix，也可以直接替换成：

```text
v1.2.1-d53f9186
```

## docker compose 示例

推荐继续使用仓库里的 `docker-compose.image.yml` 加 `.env.release.example`。

`.env` 最少需要这样配置：

```env
TZ=Asia/Shanghai
EMAIL=admin@example.com
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.2.1-d53f9186

VANBLOG_HTTP_PORT=80

POSTGRES_DB=vanblog
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
VAN_BLOG_DATABASE_URL=postgresql://postgres:postgres@postgres:5432/vanblog
VAN_BLOG_REDIS_URL=redis://redis:6379
VAN_BLOG_WALINE_DB=waline
VAN_BLOG_CADDY_MANAGE_HTTPS=false
```

## 启动方式

如果你直接用固定镜像标签：

```bash
docker compose up -d
```

如果你使用仓库里的镜像部署模板：

```bash
cp .env.release.example .env
# 把 VANBLOG_RELEASE_SUFFIX 改成 v1.2.1-d53f9186
docker compose -f docker-compose.image.yml pull
docker compose -f docker-compose.image.yml up -d
```

查看状态：

```bash
docker compose ps
docker compose logs -f caddy server website admin postgres redis
```

首次初始化入口：

```text
http://<你的域名或IP>/admin/init
```

## 升级注意事项

- 如果你之前依赖旧的 `/page/*` 深分页做导航，请同步检查是否已切到归档页入口
- 如果你使用 Cloudflare，请记得同时配置 dashboard 上的 Cache Rules，仓库里的代码只负责 origin contract 和 purge
- 如果你没有配置 `VAN_BLOG_CLOUDFLARE_API_TOKEN` / `VAN_BLOG_CLOUDFLARE_ZONE_ID`，站点仍可运行，但不会自动调用 Cloudflare purge
- 如果你希望生产环境完全可追踪，建议统一使用不可变 tag，而不是只用 `latest`

## 这版的定位

`v1.2.1` 的定位是：

- 继续稳定 PostgreSQL + Redis 架构
- 把公开页面进一步改造成 Cloudflare 边缘缓存友好结构
- 用归档路由替代深分页，降低新文章发布时的联动失效面
- 为后续继续优化评论、统计和推荐模块打基础
