# v1.6.1

`v1.6.1` 是一次面向 all-in-one 路径的 hotfix 版本：它不改博客主栈边界，不恢复 `/admin/ai`，主要解决 Waline 在 PostgreSQL 模式下打印 sqlite 误导日志的问题，同时给 `docker-compose.all-in-one.latest.yml` 增加一组更适合 8 核 / 32G / SSD 机器的默认数据库与 Redis 参数。

如果只用一句话概括：

> `v1.6.1` 修掉了 all-in-one 中 Waline 的 sqlite 启动噪音，并把 latest 单镜像部署默认值调成更适合中高配机器的性能档位。

## 本版摘要

这一版主要集中在两件事：

- 修复 Waline 在非 sqlite 部署下仍打印 sqlite 原生模块缺失堆栈的问题
- 为 `docker-compose.all-in-one.latest.yml` 增加 PostgreSQL / Redis 的默认性能参数，并让 all-in-one entrypoint 真正读取这些环境变量

## 相比 v1.6.0 的变化

如果你已经在使用 `v1.6.0`，这一版最直接的变化会出现在 all-in-one 路径：

- Waline 在 PostgreSQL 模式下不再因为尝试加载 sqlite adapter 而打印一大段误导性的 `MODULE_NOT_FOUND`
- `docker-compose.all-in-one.latest.yml` 默认就会带上数据库与 Redis 的性能参数
- `docker/all-in-one/entrypoint.sh` 不再只用 PostgreSQL / Redis 的保守默认值，而是能读取并应用这些环境变量

这不是功能扩展版本，而是一次部署体验与默认性能档位的收口版本。

## 主要更新

### 1. 收口 Waline sqlite 启动噪音

仓库新增：

- `scripts/fix-waline-adapter.js`

并在下面两个镜像构建路径中执行：

- `docker/waline.Dockerfile`
- `docker/all-in-one.Dockerfile`

修复后的行为是：

- 如果当前部署真的启用了 sqlite，仍然保留 sqlite 相关错误输出
- 如果当前部署走的是 PostgreSQL，就不再打印 sqlite 原生模块缺失堆栈

这能避免 all-in-one 用户在评论功能正常的情况下，被启动日志误导成“评论数据库坏了”。

### 2. all-in-one entrypoint 支持数据库与 Redis 调优变量

`docker/all-in-one/entrypoint.sh` 现在支持并应用下面这些环境变量：

- PostgreSQL
  - `POSTGRES_SHARED_BUFFERS`
  - `POSTGRES_WORK_MEM`
  - `POSTGRES_MAINTENANCE_WORK_MEM`
  - `POSTGRES_EFFECTIVE_CACHE_SIZE`
  - `POSTGRES_MAX_CONNECTIONS`
  - `POSTGRES_CHECKPOINT_TIMEOUT`
  - `POSTGRES_MAX_WAL_SIZE`
- Redis
  - `REDIS_SAVE_POLICY`
  - `REDIS_APPENDONLY`
  - `REDIS_MAXMEMORY`
  - `REDIS_MAXMEMORY_POLICY`

PostgreSQL 会把这些参数写入运行时的 `vanblog-tuning.conf`，Redis 会按这些值生成自己的启动配置文件。

### 3. all-in-one latest 默认值改成中高配机器档位

`docker-compose.all-in-one.latest.yml` 现在默认带上这组参数：

```env
POSTGRES_SHARED_BUFFERS=8GB
POSTGRES_WORK_MEM=32MB
POSTGRES_MAINTENANCE_WORK_MEM=1GB
POSTGRES_EFFECTIVE_CACHE_SIZE=24GB
POSTGRES_MAX_CONNECTIONS=200
POSTGRES_CHECKPOINT_TIMEOUT=15min
POSTGRES_MAX_WAL_SIZE=8GB
REDIS_SAVE_POLICY=900 1 300 10 60 10000
REDIS_APPENDONLY=yes
REDIS_MAXMEMORY=4gb
REDIS_MAXMEMORY_POLICY=allkeys-lru
```

这组默认值偏向：

- 8 核 CPU
- 32G 内存
- SSD / NVMe 磁盘
- 中小型到中型博客站点

如果别人机器更小或更大，仍然可以自行覆盖这些环境变量。

## 配置与兼容性

`v1.6.1` 不引入数据库迁移，也不改业务数据模型：

- 不需要数据库结构迁移
- 不需要重新初始化博客
- 从 `v1.6.0` 可直接升级

唯一需要注意的是：

- 只有升级到新的 `v1.6.1` all-in-one 镜像，新的 entrypoint 调优能力和 Waline adapter 修复才会真正生效

## docker compose 配置单

如果你使用锁版镜像部署：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.6.1-<image-id>
```

all-in-one latest：

```bash
docker compose -f docker-compose.all-in-one.latest.yml pull
docker compose -f docker-compose.all-in-one.latest.yml up -d
```

all-in-one 锁版：

```bash
docker compose -f docker-compose.all-in-one.image.yml pull
docker compose -f docker-compose.all-in-one.image.yml up -d
```

## 升级建议

升级到 `v1.6.1` 后，建议重点看这几个地方：

1. all-in-one 启动日志里不再出现 Waline 的 sqlite 原生模块缺失堆栈
2. 评论接口和 Waline 管理页仍然可正常使用
3. Redis 不再每分钟因为 `save 60 1` 而频繁刷盘
4. PostgreSQL 与 Redis 的运行参数已经按 latest 模板默认值生效

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.6.0-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.all-in-one.image.yml up -d
```
