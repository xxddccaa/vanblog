# v1.6.3

`v1.6.3` 是一次前台滚动体验修复版本，重点收口自定义背景图场景下的顶端越界滚动问题。

如果只用一句话概括：

> `v1.6.3` 限制前台页面到达顶部后的纵向 overscroll，避免用户继续向上滚动时页面内容被浏览器回弹拉开，只剩站点背景图。

## 本版摘要

这一版主要集中在两件事：

- 前台 `html` / `body` 增加纵向 overscroll 边界，滚动到顶部后不再继续暴露背景图
- 增加样式回归测试，确保滚动边界和移动端背景图性能回退规则不会被误删

## 相比 v1.6.2 的变化

如果你已经在使用 `v1.6.2`，这一版最直接的变化是：

- 设置了前台背景图时，鼠标滚轮或触控板在页面顶部继续向上滚动，不会再出现只剩背景图的空视口
- 首页、文章页、归档页、标签页、分类页等共用前台布局的页面都受益
- 移动端仍保留 `background-attachment: scroll`，避免固定背景图造成额外重绘

## 主要更新

### 1. 前台滚动边界收口

前台全局样式在 `html` 和 `body` 上增加：

```css
overscroll-behavior-y: none;
```

这个修复不拦截 `wheel` 事件，也不改动页面滚动容器结构，因此不会影响正常滚动、文章目录跳转、回到顶部按钮或导航栏 sticky 行为。

### 2. 自定义背景图场景更稳定

VanBlog 前台背景图由 `--bg-image` / `--bg-image-dark` 驱动，并挂在页面根背景上。过去浏览器顶端越界回弹时，内容层会被临时拉开，视觉上只剩用户配置的背景图。

本版限制根滚动边界后，页面在顶端会停止在可见内容位置，背景图仍作为正常页面背景显示。

### 3. 增加回归测试

新增前台样式测试，覆盖：

- `html` / `body` 必须保留 `overscroll-behavior-y: none`
- 移动端背景附件回退规则必须保留

## 配置与兼容性

这一版不引入数据库迁移，不改变 API、环境变量或 compose 拓扑。

如果你使用自定义 CSS 覆盖了 `html` / `body` 的 `overscroll-behavior-y`，请确认没有重新改回 `auto`，否则顶部越界回弹可能再次出现。

## docker compose 配置单

如果你使用锁版镜像部署：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.6.3-<image-id>
```

## 升级建议

升级到 `v1.6.3` 后，建议重点验证：

1. 前台已配置背景图时，首页顶部连续向上滚动不会只剩背景图
2. 文章页顶部连续向上滚动不会只剩背景图
3. 正常向下滚动、目录跳转、回到顶部按钮仍可使用
4. 移动端背景图滚动表现仍然流畅

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.6.3-<image-id>
```

改回上一版的 release suffix，然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```
