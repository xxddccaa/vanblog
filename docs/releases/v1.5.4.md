# v1.5.4

`v1.5.4` 是一次围绕前台首屏性能、Markdown 富内容渲染链路和阅读稳定性的结构性补丁发布。

如果只用一句话概括：

> `v1.5.4` 把首页和文章页从同一条重型客户端 Markdown 链路里拆开，让文章主题 CSS 在首屏就绪，避免正文先显示再被后到样式覆盖，同时把搜索弹层、统计脚本和几处无障碍细节一起收口。

## 本版摘要

这一版主要集中在四件事：

- 首页卡片和富内容文章卡片彻底拆开，首页不再为了摘要列表首屏带入整套 Markdown Viewer 运行时
- 文章页与 about 页改为服务端先产出 Markdown HTML，客户端只做代码块、Mermaid、图片放大等增强层
- Markdown 主题 CSS 从全站根布局下放到富内容路由，并在首屏 head 直出，减少样式闪烁
- 搜索弹层、统计脚本与几处无障碍问题一起收口，降低首屏干扰和 Lighthouse 的非必要扣分

## 相比 v1.5.3 的变化

如果你已经在使用 `v1.5.3`，这一版最直接的变化会体现在前台公开页面：

- 首页不再和文章页共用同一套重型客户端正文渲染路径
- 打开文章时，Markdown 主题样式会在首屏就绪，而不是 hydration 之后再追加到 `head`
- 搜索弹层不再在首页 SSR 时就带着一整块隐藏 modal 壳子
- 统计脚本会延后到空闲时或首次交互后再注入，不再抢首屏资源

这不是一次视觉换皮版本，而是一次“首屏链路拆分 + 正文渲染路径收敛 + 样式闪烁治理”的补丁发布。

## 主要更新

### 1. 首页与富内容页首屏链路正式拆分

此前首页摘要列表复用了完整文章卡片组件，结果是：

- 首页即使只显示摘要，也会静态带入 Markdown Viewer 相关依赖
- 首页首屏 JS 很容易和文章页落在同一档位
- 只想看列表的用户，也要承担正文渲染链路的下载和解析成本

`v1.5.4` 现在把这两条路径明确拆开：

- 首页改用轻量 `OverviewPostCard`
- 文章页和 about 页改用 `RichPostCard`
- 首页摘要直接输出纯文本预览，不再首屏挂载完整正文查看器

这意味着首页不再为了文章摘要背上整条客户端 Markdown 运行时链路。

### 2. 文章正文改成服务端 HTML + 客户端增强层

这一版把公开文章和 about 页的正文渲染改成了两段式：

- 服务端先按现有 Markdown 规则产出 HTML
- 客户端只接管必要的增强能力

保留的增强能力包括：

- 代码块复制、折叠与自动换行
- 标题锚点绑定
- Mermaid 渲染与导出
- 图片放大

这样做的结果是：

- 正文首屏内容更早可见
- 公共文章默认不再依赖完整客户端 Viewer 才能把正文画出来
- 私密文章仍保留解锁后的客户端兜底路径，不破坏已有使用方式

### 3. Markdown 主题 CSS 改成富内容路由首屏直出

你之前遇到的“文章先按一套样式显示，几秒后又被另一套样式覆盖”的问题，本质上就是主题 CSS 在客户端后插入导致的样式闪烁。

`v1.5.4` 的处理方式是：

- Markdown 主体 CSS 只在富内容路由加载，不再挂在全站根布局
- 文章页和 about 页继续在首屏 head 直出 Markdown 主题 CSS
- 客户端布局层只保留兼容性同步逻辑，不再默认改变首屏顺序

这会明显降低文章页的 FOUC / 样式重算体感，也避免首页无意义地加载整套 Markdown 主题样式。

### 4. 搜索弹层和统计脚本改成按需 / 延后加载

这一版还顺手把两个首屏噪音点处理了：

- 搜索弹层改成首次打开时再动态挂载，不再始终 SSR 一个隐藏 modal
- GA / 百度统计改成空闲时或首次交互后再注入，不再优先抢首页首屏资源

这不会改变后台配置方式，但会减少首页初始 HTML 和关键路径上的不必要负担。

### 5. 无障碍与稳定性快修一起落地

这一版同时补了几处前台细节问题：

- `viewport` 去掉了 `user-scalable=no`
- 主内容区域补齐 `main` 和 `aside` 语义
- 页面顶部增加“跳到正文” skip link
- 邮箱社交入口从假链接改成真正的按钮
- 备案图补齐显式高度，减少 Lighthouse 的图片尺寸告警

这些都不是大功能，但属于会持续影响真实浏览与自动化检查的基础项。

## 配置与兼容性

`v1.5.4` 不引入新的环境变量，也不需要数据库迁移：

- 不新增 `.env` 配置项
- 不需要数据结构迁移
- `ByteMD` 仍然保持为默认编辑器
- 从 `v1.5.3` 可直接升级

需要注意的一点是：

- 如果你的站点当前没有配置 `markdownLightThemeUrl` / `markdownDarkThemeUrl`，那么这次“富内容页首屏直出 Markdown 主题 CSS”的能力仍然会按空配置运行，不会自动帮你补主题地址

## 发布与部署边界

`v1.5.4` 继续保持当前多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 仍然使用官方镜像
- AI 工作台继续随 `server` / `admin` 核心镜像发布
- bundled FastGPT 仍然是可选叠加，不属于默认主栈公开拓扑

所以这次升级仍然是“主栈正常升级，AI 按需叠加”。

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.5.4-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级建议

升级到 `v1.5.4` 后，建议重点看这几个地方：

1. 首页是否仍然首屏快速展示摘要，且不再出现富内容页那种重型渲染延迟
2. 文章页和 about 页在首次打开时是否还会出现主题样式后覆盖的闪烁
3. 搜索弹层是否只在首次触发时挂载，而不是首页源码里始终带着隐藏 modal
4. 统计脚本是否仍能正常注入，但不再优先出现在首屏关键路径
5. `Ctrl/Cmd + K` 搜索、代码块增强、Mermaid、图片放大和私密文章解锁是否都保持正常

如果看到的仍然是旧样式或旧资源，优先强刷浏览器缓存和 CDN 缓存再确认。

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.3-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只是主题样式看起来没变化，优先排查浏览器缓存、CDN 缓存或站点当前 Markdown 主题配置，而不是先判断镜像回滚失败。
