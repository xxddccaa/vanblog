# v1.3.7

`v1.3.7` 是在 `v1.3.6` 基础上，围绕 ByteMD 编辑体验、前台 Markdown 主题表现、后台品牌配置与后台侧边栏交互做的一次综合补丁发布。

如果只用一句话概括这个版本：

> `v1.3.7` 重点把“后台编辑区有颜色、预览区和前台发灰/发黑/不一致”的那一批 Markdown 体验问题整体收口，同时补齐后台独立 favicon、代码块换行与展开、Mermaid 对齐，以及后台侧边栏在暗色与响应式场景下的可用性。

## 本版摘要

本次版本主要集中在四个方向：

- 补齐 ByteMD 在后台编辑区、预览区和前台三端的 Markdown 主题一致性与彩色表现
- 修复暗色主题下代码块、Mermaid、标题样式和长代码阅读体验
- 为后台增加独立 favicon 配置，并补齐 `/admin` 品牌资源应用链路
- 调整后台侧边栏 footer 区域、主题切换和响应式收拢逻辑，提升窄屏与暗色使用体验

## 相比 v1.3.6 的变化

如果你当前线上或外部分发仍以 `v1.3.6` 为基线，那么 `v1.3.7` 相比 `v1.3.6` 可以重点关注下面这些变化：

### 1. Markdown 不再只是“编辑区有颜色”，预览区和前台也一起跟上

此前后台编辑器存在明显割裂：

- 编辑区本身有一定语法颜色
- 预览区和前台文章却经常偏黑白
- 暗色模式下部分标题、代码块、Mermaid 文本对比还不够

`v1.3.7` 这次把 Markdown 主题能力从“能选主题”推进到“前后台真正统一”：

- 后台新增 Markdown 主题配置解析工具，亮色 / 暗色主题都支持 preset 或自定义 CSS
- 前台布局会稳定读取并注入对应 Markdown 主题链接
- 后台编辑器预览区也会跟随同一套主题配置加载
- 多个主题下过于突兀的二级标题描边 / 环绕视觉被单独软化

这意味着：

- 你在 `/admin/site/setting?...mdTheme` 里调的 Markdown 主题，前台和预览区都能更一致地体现
- 不再出现“编辑区有设计感，预览区/前台像没上主题”的割裂感

### 2. 暗色模式下的代码块和 Mermaid 终于更稳定可读

这一版还集中处理了一批在暗色模式下最影响阅读体验的问题：

- 后台预览区未标注语言的 fenced code block 会落成黑字，几乎看不清
- 长代码块缺少明显的横向滚动与自动换行按钮，阅读成本高
- 前台 Mermaid 不居中，而预览区是居中的
- 后台 Mermaid 在暗色模式下文字发白、对比不足、布局偏移

`v1.3.7` 的处理方式是整条链路一起修：

- 代码块 header 统一补齐“复制 / 自动换行 / 展开代码”交互
- 代码块最大显示行数接入站点设置，不再硬编码 15 行
- 未高亮代码块在暗色主题下增加浅色文本兜底
- 预览区与前台的 Mermaid 居中、文本颜色和容器样式重新对齐

如果你的文章里有 `.env`、命令片段、配置行、Mermaid 流程图，这版会比 `v1.3.6` 明显更稳。

### 3. 后台现在可以单独配置 favicon，不再和前台强绑定

`v1.3.6` 修的是前台 favicon 生效问题；`v1.3.7` 继续把后台品牌链路补齐。

现在后台支持独立的 `adminFavicon`：

- 如果没有上传后台图标，默认继续回退到 `/admin/logo.svg`
- 如果上传了后台图标，`/admin` 页面的 favicon 会优先使用后台独立图标
- 公开接口和后台初始化链路都补齐了 `adminFavicon` 字段透传

这让“前台站点品牌”和“后台管理品牌”终于可以分开维护，不再共享一个图标入口。

### 4. 后台侧边栏 footer 和响应式行为更像一个完整产品了

此前后台左下角 footer 区域有几类体感问题：

- `主站 / 关于 / 暗色模式 / 登出` 与上方菜单风格不一致
- 收拢后下方图标与上方图标表现不统一
- 在接近 `992px` 的宽度下会出现让人迷惑的自动收拢

`v1.3.7` 做了下面这些调整：

- footer 改成统一的菜单式结构
- 主题按钮文案收成 `亮色 / 暗色`
- desktop 收拢偏好写入 `localStorage`
- 自动响应式切换改成只在真正窄屏下触发，避免中等宽度误收拢
- 编辑页面里会干扰侧边栏状态的逻辑也一起清掉

## 主要特性 / 修复

### Markdown / 编辑器体验

- 后台编辑器预览区支持按站点设置加载 Markdown 主题 CSS
- 前台补齐 Markdown 主题解析与默认主题回退逻辑
- 新增多套亮色 / 暗色 Markdown 主题资源，并对部分主题的二级标题视觉做细化调整
- 修复暗色主题下未标注语言的代码块在后台预览区显示为黑字的问题
- 代码块统一支持复制、自动换行、展开/收起，并保留横向滚动能力
- 代码块折叠行数接入站点设置中的“代码块最大显示行数”

### Mermaid / 预览展示

- 修复后台编辑器预览区 Mermaid 不够居中、文本发白和对比不足的问题
- 修复前台文章页 Mermaid 未居中的问题
- 统一 Mermaid 容器画布、内边距、边框与暗色文本策略

### 前台布局 / 自适应

- 前台文章内容区域支持新的铺满布局选项，而不是只铺满背景层
- `LayoutBody` 与文章页布局逻辑调整，真正让正文区域根据配置扩展宽度
- 增加相关前台布局与 Markdown 主题回归测试

### 后台品牌与导航

- 后台站点设置新增独立 `adminFavicon` 上传能力
- `/admin` 页面初始化时会应用后台独立 favicon，并保留默认回退
- 侧边栏 footer 重构为统一菜单，包含 `主站 / 关于 / 亮色|暗色 / 登出`
- 窄屏下改为移动端菜单抽屉形态，烟测同步覆盖

## 部署前注意事项

发布 `v1.3.7` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.7`
- Git tag 使用 `v1.3.7`
- Docker Hub 推送使用本机 `pnpm release:images:push`
- 当前正式发布仍默认只推 `linux/amd64`

如果你的站点最近遇到下面任意现象，这版建议优先升级：

- 后台编辑器预览区代码块在暗色模式下发黑、看不清
- 前台文章中的 Mermaid 没有居中
- 后台和前台共用一个 favicon，无法区分品牌
- Markdown 主题在编辑区、预览区、前台三端表现不一致

## docker compose 配置单

推荐生产或分发场景优先使用仓库自带的：

- `docker-compose.image.yml`
- `.env.release.example`

如果你需要给别人一份可以直接改变量部署的示例，下面是一份最小可用 `docker-compose.yaml`。这里直接使用版本别名 tag。

## 升级步骤

- 先备份当前生产环境的 `data/`、`log/`、`caddy/` 目录
- 把部署侧的镜像版本切到 `v1.3.7` 或对应不可变 tag
- 执行 `docker compose -f docker-compose.image.yml up -d`
- 升级后重点验证：
  - `/admin` 是否使用新的后台 favicon
  - 后台编辑器预览区暗色代码块是否正常可读
  - 前台文章页 Mermaid 是否居中
  - 前台 Markdown 主题和代码块折叠/换行是否符合预期

## 回滚方法

如果你在 `v1.3.7` 升级后需要快速回滚：

- 直接把 `docker-compose.image.yml` 或部署环境变量中的版本别名 tag 改回 `v1.3.6`
- 如果你需要严格锁定单次构建结果，则改回对应的不可变 tag
- 回滚后建议清理浏览器缓存并重新打开 `/admin` 与前台文章页，避免旧的主题 CSS 与 favicon 资源残留
