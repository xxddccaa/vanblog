# v1.5.3

`v1.5.3` 是一次围绕 `/admin/document` 私密文档页的体验重构补丁，同时把后台与前台代码块的暗色高亮进一步收口到石墨黑语义。

如果只用一句话概括：

> `v1.5.3` 把私密文档从“树 + 多层卡片预览”重构成更像知识库的统一工作区，并顺手把移动端阅读链路、空状态文案和代码块暗色配色一起收干净。

## 本版摘要

这一版主要集中在三件事：

- `/admin/document` 改成统一工作区，文档树和预览区不再像两张彼此独立的卡片
- 移动端私密文档页改成更轻的工具条 + 单阅读面板，减少重复按钮和壳层噪音
- 后台与前台代码块暗色高亮继续从蓝青感收口，和 `v1.5.2` 的石墨黑主题更一致

## 相比 v1.5.2 的变化

如果你已经在用 `v1.5.2`，这一版最直接的感受会是：

- 私密文档页不再是“左边树、右边又套一层层卡片”的旧布局
- 空状态、库概览、文档预览现在都挂在同一个阅读工作区里
- 手机端进入私密文档后，顶部壳层更轻，操作入口也更少但更清楚
- 代码块深色语义进一步去掉蓝青味，后台预览和前台文章看起来更统一

这依然不是功能扩张版本，更像一次“私密文档工作流重做 + 暗色样式继续收口”的补丁发布。

## 主要更新

### 1. 私密文档页重构为统一工作区

`/admin/document` 这次不再维持原来那种“树一块、预览一块、空态再套一块”的感觉，而是改成统一工作区：

- 左侧是文档树 explorer
- 右侧是当前上下文的阅读区
- 桌面端支持收起 / 展开文档树
- `Ctrl/Cmd + B` 继续保留为快速切换快捷键

这样做的结果是：

- 浏览、搜索、预览终于落在同一条操作链路里
- 页面不再显得像几个不相关容器拼在一起
- 选中文档和未选择文档的状态切换也更自然

### 2. 文档树交互与密度重新收口

文档树这版不只换壳，也把交互细节一起收了：

- 节点选中态改成更轻的左侧强调线 + 柔和底色
- hover 时再显示操作按钮，减少常驻噪音
- 文档树缩进、节点高度、图标和操作按钮尺寸都更紧凑
- 文档库和文档的层级关系更清楚，但不会再显得松散

移动端下，文档树仍保留足够点击面积，不会因为收紧密度而变得难点。

### 3. 空状态与文档库概览融入同一阅读面板

私密文档页原来最突兀的地方之一，是空状态和库概览很容易再叠出一层“额外色块”。

`v1.5.3` 继续沿着 `v1.5.2` 的方向收口：

- “请选择文档” 空状态直接作为阅读区内容出现
- 文档库概览不再额外强调成另一张重卡片
- 辅助文案保持在当前背景里，用文字层级区分，而不是再塞一个盒子

这会让私密文档页更像“内容工作区”，而不是“卡片套卡片的后台页”。

### 4. 移动端私密文档页改成更轻的工具条

这一版手机端私密文档页也单独做了收口：

- 顶部改成更轻的工具条，而不是再额外叠一张显眼的卡片
- 当前文档上下文会直接显示在工具条里
- 阅读区头部不再重复出现一组“打开文档树”入口
- 整体层级更少，操作仍保留在最短路径里

如果你之前觉得移动端像“壳层太多”，这一版会明显更干净。

### 5. 后台 / 前台代码块暗色高亮继续改成石墨黑语义

这次还顺手把代码块深色高亮继续收了收：

- 一批偏蓝青的高亮色被替换成更中性的石墨黑系语义
- 后台预览和前台文章代码块看起来更像同一套语言
- 不会再出现整体背景已经很克制，但代码一打开又明显偏蓝青的割裂感

这部分没有引入新的主题切换逻辑，主要是直接校准现有暗色高亮颜色。

## 配置与兼容性

`v1.5.3` 不引入新的环境变量，也不需要数据库迁移：

- 不新增 `.env` 配置项
- 不需要数据结构迁移
- `ByteMD` 仍然保持为默认编辑器
- 从 `v1.5.2` 直接升级即可

## 发布与部署边界

`v1.5.3` 继续保持当前多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 仍然使用官方镜像
- AI 工作台继续随 `server` / `admin` 核心镜像发布
- bundled FastGPT 仍然是可选叠加，不属于默认主栈公开拓扑

所以这次升级仍然是“主栈正常升级，AI 按需叠加”。

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.5.3-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级建议

升级到 `v1.5.3` 后，建议重点看这几个地方：

1. `/admin/document` 是否已经变成统一工作区，而不是旧的多卡片预览结构
2. 桌面端文档树收起 / 展开和 `Ctrl/Cmd + B` 是否正常
3. 移动端私密文档页顶部是否已经改成更轻的工具条，且不再出现重复入口
4. 空状态、文档库概览、纯预览页是否都已经融入当前背景
5. 前台和后台代码块在暗色模式下是否还残留明显蓝青感

如果看到的仍然是旧样式，优先强刷浏览器缓存和 CDN 缓存再确认。

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.2-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只是界面看起来没变，优先排查浏览器缓存、CDN 缓存或拉取到的镜像 tag 是否正确，而不是先判断回滚失败。
