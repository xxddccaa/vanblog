# v1.3.4

`v1.3.4` 是在 `v1.3.3` 基础上继续补齐镜像部署默认体验的一次补丁版发布。

如果只用一句话概括这个版本：

> `v1.3.4` 重点解决了 Waline 共享密钥不该依赖人工填写的问题，让镜像在首次启动时自动生成并持久化这份密钥，同时补上了一个更适合直接落地的 `latest` 部署模板。

## 本版摘要

本次版本主要集中在三个方向：

- 让 Waline 与内部控制面的共享 JWT 在启动时自动生成并落盘
- 让 `server`、`website`、`waline` 在镜像部署里稳定复用同一份共享密钥
- 增加 `docker-compose.latest.yml`，方便直接使用当前目录结构拉取 `latest`

## 相比 v1.3.3 的变化

如果你当前线上或外部分发仍然以 `v1.3.3` 为基线，那么 `v1.3.4` 相比 `v1.3.3` 可以重点关注下面这些变化：

### 1. Waline 共享密钥不再依赖人工记忆

此前镜像部署里，`WALINE_JWT_TOKEN` 需要用户自己手动填写。这个机制本身没有错，但在真实部署里很容易出现：

- 忘记填写
- 随手保留占位符
- `server`、`website`、`waline` 三处值不一致

`v1.3.4` 改成了更稳妥的方式：

- 首次启动时自动生成一份共享密钥
- 把它写入日志目录中的 `waline.jwt`
- 后续重启继续复用同一份密钥

如果你仍然想显式指定 `WALINE_JWT_TOKEN`，当前版本依然支持；但默认路径已经不再要求人工先准备这串值。

### 2. `server` / `website` / `waline` 现在共享同一份落盘密钥

这次不是只让 `waline` 容器自己生成随机值，而是让整条控制链路共享同一份持久化密钥：

- `server` 启动前先确保共享密钥存在
- `website` 启动前先确保共享密钥存在
- `waline` 启动前先确保共享密钥存在

这样能保证：

- Waline 自身 JWT
- VanBlog 到 Waline 的控制请求
- Website 相关控制/回退令牌链路

在默认镜像部署中都保持一致，不再因为人工漏填环境变量而失配。

### 3. 新增 `docker-compose.latest.yml`

很多人只是想在另一台机器上快速复用当前目录结构，并不想维护 `docker-compose.image.yml + .env.release.example + VANBLOG_RELEASE_SUFFIX` 这一整套变量。

`v1.3.4` 直接补了一个简化入口：

- `docker-compose.latest.yml`

这个文件的特点是：

- 默认拉取 `kevinchina/deeplearning:vanblog-*-latest`
- 使用当前目录下的 `./data`、`./log`、`./caddy` 等挂载路径
- 继续保留 Redis 健康检查、Waline 服务和完整拓扑

它更适合个人维护或快速部署场景；如果你要严格锁定某次发布，仍然优先使用 `docker-compose.image.yml`。

## 主要特性 / 修复

### Waline 与控制面

- 新增运行时脚本，在镜像启动时自动生成并持久化共享 Waline JWT
- 共享密钥默认写入日志目录中的 `waline.jwt`
- `server`、`website`、`waline` 启动链路改为统一复用同一份密钥

### Docker 部署模板

- 新增 `docker-compose.latest.yml`
- `docker-compose.yml`、`docker-compose.image.yml`、`docker-compose.latest.yml` 一起补齐日志挂载与共享密钥逻辑
- release / deploy 文档改成允许 `WALINE_JWT_TOKEN` 留空并自动生成

### 部署文档

- `.env.release.example` 中的 `WALINE_JWT_TOKEN` 改为可留空
- `DEPLOY.md` 补充 `waline.jwt` 的说明
- `docs/reference/env.md`、`docs/guide/get-started.md` 同步更新

## 部署前注意事项

发布 `v1.3.4` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.4`
- Git tag 使用 `v1.3.4`
- Docker Hub 推送使用本机 `pnpm release:images:push`
- 生产部署若要求精确回滚，仍优先使用不可变镜像标签

如果你使用镜像部署并准备沿用自动密钥机制，请确认：

- `server`、`website`、`waline` 都能访问同一份日志目录挂载
- 不要删除 `log/waline.jwt`，除非你明确希望轮换 Waline 共享密钥

## docker compose 配置单

推荐生产或分发场景优先使用仓库自带的：

- `docker-compose.image.yml`
- `.env.release.example`

如果你需要给别人一份可以直接改变量部署的示例，下面是一份最小可用 `docker-compose.yaml`。这里直接使用版本别名 tag，不再要求额外填写镜像 id。

```yaml
services:
  caddy:
    image: kevinchina/deeplearning:vanblog-caddy-v1.3.4
    restart: always
    depends_on:
      server:
        condition: service_healthy
      website:
        condition: service_healthy
      admin:
        condition: service_healthy
      waline:
        condition: service_healthy
    environment:
      TZ: Asia/Shanghai
      EMAIL: your-email@example.com
    volumes:
      - ./log:/var/log
      - ./caddy/config:/root/.config/caddy
      - ./caddy/data:/root/.local/share/caddy
    ports:
      - "80:80"

  server:
    image: kevinchina/deeplearning:vanblog-server-v1.3.4
    restart: always
    init: true
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      TZ: Asia/Shanghai
      NODE_ENV: production
      VAN_BLOG_DATABASE_URL: postgresql://postgres:postgres@postgres:5432/vanblog
      VAN_BLOG_REDIS_URL: redis://redis:6379
      VAN_BLOG_WALINE_DB: waline
      VAN_BLOG_WALINE_DATABASE_URL: postgresql://postgres:postgres@postgres:5432/waline
      VAN_BLOG_CADDY_MANAGE_HTTPS: "false"
      VANBLOG_CADDY_API_URL: http://caddy:2019
      VANBLOG_WEBSITE_CONTROL_URL: http://website:3011
      VANBLOG_WALINE_CONTROL_URL: http://waline:8361
      VANBLOG_WEBSITE_ISR_BASE: http://website:3001/api/revalidate?path=
      WALINE_JWT_TOKEN: ""
    volumes:
      - ./data/static:/app/static
      - ./log:/var/log
      - ./aliyunpan/config:/root/.config/aliyunpan
    expose:
      - "3000"
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O /dev/null http://127.0.0.1:3000/swagger || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 30s

  website:
    image: kevinchina/deeplearning:vanblog-website-v1.3.4
    restart: always
    init: true
    environment:
      TZ: Asia/Shanghai
      NODE_ENV: production
      PORT: 3001
      WEBSITE_CONTROL_PORT: 3011
      VAN_BLOG_SERVER_URL: http://server:3000
      VAN_BLOG_REVALIDATE: "false"
      VAN_BLOG_ALLOW_DOMAINS: pic.mereith.com
      WALINE_JWT_TOKEN: ""
    expose:
      - "3001"
      - "3011"
    volumes:
      - ./log:/var/log
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O - http://127.0.0.1:3011/health >/dev/null || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 20s

  admin:
    image: kevinchina/deeplearning:vanblog-admin-v1.3.4
    restart: always
    expose:
      - "3002"
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O /dev/null http://127.0.0.1:3002/admin/ || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 20s

  waline:
    image: kevinchina/deeplearning:vanblog-waline-v1.3.4
    restart: always
    init: true
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      TZ: Asia/Shanghai
      NODE_ENV: production
      PORT: 8360
      WALINE_CONTROL_PORT: 8361
      PG_HOST: postgres
      PG_PORT: 5432
      PG_DB: waline
      PG_USER: postgres
      PG_PASSWORD: postgres
      JWT_TOKEN: ""
    expose:
      - "8360"
      - "8361"
    volumes:
      - ./log:/var/log
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O - http://127.0.0.1:8361/health >/dev/null || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 30s

  postgres:
    image: postgres:16-alpine
    restart: always
    environment:
      TZ: Asia/Shanghai
      POSTGRES_DB: vanblog
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    volumes:
      - ./data/postgres:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres -d vanblog"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 20s

  redis:
    image: redis:7-alpine
    restart: always
    volumes:
      - ./data/redis:/data
    command: ["redis-server", "--appendonly", "yes"]
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 10s
```
