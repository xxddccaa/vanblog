# v1.0.3

这是旧版 MongoDB 架构下的稳定导出桥接版本。

如果你现在的线上博客还是老架构，推荐先用 `v1.0.3` 启动旧版博客，进入后台导出 JSON，再迁移到 `v1.1.0`。

## 适用场景

- 你的旧博客数据还在 MongoDB 数据目录里
- 你需要一个“能正常导出 JSON”的旧版镜像
- 你准备后续迁移到 PostgreSQL + Redis 新版本

## 最推荐的迁移方式

1. 用 `v1.0.3` 启动旧版站点
2. 访问后台 `/admin/site/setting?tab=backup`
3. 点击“导出站点 JSON”
4. 保存导出的 JSON 文件
5. 再部署 `v1.1.0`，把这份 JSON 导入新系统

## 示例 docker-compose.yaml

下面是一个最小可用示例。请把它保存成 `docker-compose.yaml` 后执行 `docker compose up -d`。

      server:

        condition: service_healthy
      admin:
        condition: service_healthy

        condition: service_healthy

      TZ: Asia/Shanghai

## 使用说明

- 如果你已经有旧版 `./data/mongo` 数据目录，直接挂进去即可
- 如果你的旧环境一直固定使用其他 Mongo 版本，请尽量保持和旧环境一致，不要随意降级数据目录
- 访问地址示例：`http://<服务器IP>:18080/admin`
- 导出路径：`/admin/site/setting?tab=backup`

## 这版的定位

`v1.0.3` 不是新的长期架构版本，它最重要的作用是：

- 保持旧版博客内容可读
- 正常导出完整站点 JSON
- 为迁移到 `v1.1.0` 提供稳定中转
