# v1.3.12

`v1.3.12` 是在 `v1.3.11` 基础上，围绕后台管理端在手机浏览器上的整体可用性、页面协调感和深浅色一致性做的一次正式补丁发布。

如果只用一句话概括这个版本：

> `v1.3.12` 重点解决“admin 在手机上能打开，但很多页面看起来仍像桌面后台硬压到小屏里，层级重复、留白不协调、操作区拥挤”这一批已经影响日常管理和验收的问题。

## 本版摘要

本次版本主要集中在四个方向：

- 为后台补上一套共享的移动端管理壳层，包括更统一的页边距、卡片节奏、移动操作区、抽屉、分页和深色模式表面层
- 欢迎页、文章、草稿、动态、私密文档、思维导图、日志、流水线、导航、自定义页面、数据管理、评论管理、系统配置等后台页面，统一补齐手机端布局和卡片化管理模式
- 针对 `动态管理`、`私密文档`、`思维导图` 与脑图编辑器继续做第二轮收口，减少标题重复、桌面式空白和多层页头造成的视觉负担
- 后台侧边栏、页面 tabs、筛选区和按钮密度进一步响应式调整，让手机 / 平板打开 `/admin` 时更接近真正可用的移动管理端，而不是只“勉强能看”

## 相比 v1.3.11 的变化

如果你当前线上或镜像分发仍以 `v1.3.11` 为基线，那么 `v1.3.12` 相比 `v1.3.11` 建议重点关注下面这些变化：

### 1. 后台现在有了一套真正统一的手机端管理壳层

`v1.3.11` 之前，后台虽然已经有部分页面在手机上可访问，但整体仍有几个明显问题：

- 页面之间的边距、卡片密度和按钮节奏不一致
- 有些页面保留桌面式大表格，有些页面改成卡片，视觉语言不统一
- 页头、标题、副标题和页面内说明经常重复出现，看起来像多层壳子叠在一起
- 抽屉、分页、tabs、操作区在手机端缺少一套共享规则

`v1.3.12` 补上了这套共享移动壳层：

- 统一移动端内容边距和页面容器留白
- 提供统一的移动卡片列表、移动 tabs 和移动操作区布局
- 补齐按钮最小高度、分页、抽屉、空态和深色主题下的 surface token
- 后台侧边栏在窄屏下更稳定地收起 / 展开，减少内容区被挤压

对实际使用来说，这意味着手机端后台不再是“每个页面各自凑合”，而是开始具备统一的操作节奏。

### 2. 常用后台列表页基本都切到了手机优先的卡片管理模式

这一版把后台高频页面进一步统一到了手机可管理的形态，包括：

- 欢迎页分析概览
- 文章管理
- 草稿管理
- 动态管理
- 私密文档
- 思维导图
- 日志管理
- 流水线
- 导航管理
- 自定义页面
- 评论管理
- 数据管理
- 系统配置中的多个管理 tab

典型变化是：

- 手机端优先展示搜索、筛选、主动作和关键元信息
- 把桌面表格改造成卡片列表或精简的移动版结构
- 把多列操作改造成更适合手指点击的主次按钮区
- 保留桌面端原有管理效率，不强行让桌面跟手机完全共用一套样式

这样处理后，手机端查看后台列表、做轻量编辑或进入深度页面的路径会明显更顺。

### 3. 动态管理、私密文档、思维导图三块手机端视觉继续收口

这三个页面是本轮第二次重点收口的区域。

#### 动态管理

- 去掉了桌面端多余的 `管理个人动态内容` 副标题
- 手机端只保留一套主标题，不再出现页头和内容区双重 `动态管理`
- 顶部改成统一说明 + 统计 + 主动作布局
- 动态正文在手机卡片里默认做预览裁切，避免单条内容把整页撑爆

#### 私密文档

- 手机端不再重复显示页头层和内容层两套 `私密文档`
- 文档树抽屉改成更接近全屏的移动抽屉，节点点击区和更多操作更适合触摸
- 未选中文档时有明确空态；选中文档后有上下文卡片和更清晰的进入编辑入口
- 文档预览和文档库说明区减少桌面式空白，整体更紧凑

#### 思维导图

- 手机端列表页从桌面表格切到卡片列表
- 搜索、新建、进入编辑统一放进同一套移动操作壳层
- 脑图编辑页压缩外围页头，把更多高度让给实际画布
- 手机端移除多余标题层，保留一套清晰的主标题和操作区

最终效果是：这三块页面在手机上看起来更像同一个后台，而不是三个各自独立的样式实验。

### 4. 欢迎页和后台整体协调感也一起提升了

除了具体业务页面，这一版还顺手继续收口了欢迎页和后台整体风格：

- Welcome 各 tab 在手机端切换更稳定，卡片密度和概览区域更统一
- 后台亮 / 暗主题下的表面层、边框、说明文字和内容文字对比更一致
- 移动端页头与页面内容之间的间距收紧，减少“打开就是大片无效留白”
- 管理页面之间更接近一套完整产品，而不是只修某几个局部页面

## 主要特性 / 修复

### 后台移动端壳层与通用组件

- 新增 / 补齐共享响应式能力与移动端管理组件
- 统一后台移动端分页、tabs、hero 卡、操作区、空态和卡片列表风格
- 统一手机与平板断点下的页面边距、按钮高度和内容区容器行为
- 继续收口侧边栏折叠与窄屏布局同步逻辑

### 后台页面级适配

- 欢迎页在手机端做了整体重排与 tab 呈现优化
- 文章、草稿、动态、思维导图等列表页统一为更适合手机管理的结构
- 私密文档补齐移动抽屉、上下文卡片和预览流线
- 流水线、日志、导航、自定义页面、评论管理、数据管理、系统配置等页补齐移动端卡片化和关键操作可达性

### 视觉与交互细节

- 去掉多余页头、副标题和重复标题，降低页面层级噪音
- 深色模式下统一移动卡片、抽屉、说明文字和内容文字的对比关系
- 调整移动端按钮网格和操作区密度，减少拥挤和误触
- 长文本内容默认在卡片中预览裁切，避免列表页过长

### 验证与回归

- 后台构建和测试环境手机视口已完成实际验收
- 继续维护当前 compose 下 `/admin` 的访问与静态资源加载稳定性
- 本轮改动后已更新代码图谱，便于后续继续扩展后台移动端

## 部署前注意事项

发布 `v1.3.12` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.12`
- Git tag 使用 `v1.3.12`
- Docker Hub 推送仍以本机 `pnpm release:images:push` 为准
- 当前正式发布仍默认只推 `linux/amd64`

如果你的站点最近遇到下面任意现象，这版建议优先升级：

- admin 在手机浏览器里虽然能打开，但列表页和详情页明显像桌面布局硬压缩
- 动态管理、私密文档、思维导图在手机上标题重复、留白不协调、操作位置难找
- 想在手机上做轻量后台管理，但文章 / 草稿之外的页面仍然不够顺手
- 深色模式下后台某些管理页在手机上层次发灰、边界不清晰

## docker compose 配置单

推荐生产或分发场景优先使用仓库自带的：

- `docker-compose.image.yml`
- `.env.release.example`

如果你需要给别人一份可以直接改变量部署的示例，下面是一份最小可用 `docker-compose.yaml`。这里直接使用版本别名 tag。

```yaml
services:
  caddy:
    image: kevinchina/deeplearning:vanblog-caddy-v1.3.12
    restart: always
    depends_on:
      server:
        condition: service_healthy
      website:
        condition: service_healthy
      admin:
        condition: service_healthy
      waline:
        condition: service_healthy
    environment:
      TZ: Asia/Shanghai
      EMAIL: your-email@example.com
    volumes:
      - ./log:/var/log
      - ./caddy/config:/root/.config/caddy
      - ./caddy/data:/root/.local/share/caddy
    ports:
      - "80:80"

  server:
    image: kevinchina/deeplearning:vanblog-server-v1.3.12
    restart: always
    init: true
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      TZ: Asia/Shanghai
      NODE_ENV: production
      VAN_BLOG_DATABASE_URL: postgresql://postgres:postgres@postgres:5432/vanblog
      VAN_BLOG_REDIS_URL: redis://redis:6379
      VAN_BLOG_WALINE_DB: waline
      VAN_BLOG_WALINE_DATABASE_URL: postgresql://postgres:postgres@postgres:5432/waline
      VAN_BLOG_CADDY_MANAGE_HTTPS: "false"
      VANBLOG_CADDY_API_URL: http://caddy:2019
      VANBLOG_WEBSITE_CONTROL_URL: http://website:3011
      VANBLOG_WALINE_CONTROL_URL: http://waline:8361
      VANBLOG_WEBSITE_ISR_BASE: http://website:3001/api/revalidate?path=
      WALINE_JWT_TOKEN: ""
    volumes:
      - ./data/static:/app/static
      - ./log:/var/log
      - ./aliyunpan/config:/root/.config/aliyunpan
    expose:
      - "3000"
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O /dev/null http://127.0.0.1:3000/swagger || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 30s

  website:
    image: kevinchina/deeplearning:vanblog-website-v1.3.12
    restart: always
    init: true
    environment:
      TZ: Asia/Shanghai
      NODE_ENV: production
      PORT: 3001
      WEBSITE_CONTROL_PORT: 3011
      VAN_BLOG_SERVER_URL: http://server:3000
      VAN_BLOG_REVALIDATE: "false"
      VAN_BLOG_ALLOW_DOMAINS: pic.mereith.com
      WALINE_JWT_TOKEN: ""
    expose:
      - "3001"
      - "3011"
    volumes:
      - ./log:/var/log
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O - http://127.0.0.1:3011/health >/dev/null || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 20s

  admin:
    image: kevinchina/deeplearning:vanblog-admin-v1.3.12
    restart: always
    expose:
      - "3002"
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O /dev/null http://127.0.0.1:3002/admin/ || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 20s

  waline:
    image: kevinchina/deeplearning:vanblog-waline-v1.3.12
    restart: always
    init: true
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      TZ: Asia/Shanghai
      NODE_ENV: production
      PORT: 8360
      WALINE_CONTROL_PORT: 8361
      PG_HOST: postgres
      PG_PORT: 5432
      PG_DB: waline
      PG_USER: postgres
      PG_PASSWORD: postgres
      JWT_TOKEN: ""
    expose:
      - "8360"
      - "8361"
    volumes:
      - ./log:/var/log
    healthcheck:
      test: ["CMD-SHELL", "wget -q -O - http://127.0.0.1:8361/health >/dev/null || exit 1"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 30s

  postgres:
    image: postgres:16-alpine
    restart: always
    environment:
      TZ: Asia/Shanghai
      POSTGRES_DB: vanblog
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    volumes:
      - ./data/postgres:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres -d vanblog"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 20s

  redis:
    image: redis:7-alpine
    restart: always
    volumes:
      - ./data/redis:/data
    command: ["redis-server", "--appendonly", "yes"]
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 15s
      timeout: 10s
      retries: 10
      start_period: 10s
```
