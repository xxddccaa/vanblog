# v1.3.8

`v1.3.8` 是在 `v1.3.7` 基础上，继续围绕暗色主题阅读体验、Markdown 配色一致性、前台卡片背景变量化，以及移动端实际浏览流畅度做的一次完整体验补丁发布。

如果只用一句话概括这个版本：

> `v1.3.8` 重点解决“后台编辑器、预览区、前台文章页虽然都支持暗色，但颜色体系割裂、局部文字发黑、代码块交互不优雅、手机端滚动发卡”的那一整批问题，并把前台卡片背景正式做成站点可配置能力。

## 本版摘要

本次版本主要集中在四个方向：

- 重做后台编辑器、预览区、前台文章页的暗色主题配色，让 ByteMD / Markdown / Admin Shell 形成统一的蓝青系视觉语言
- 补齐代码块在预览区和前台的交互与可读性，包括自动换行、展开收起、未标注语言代码块暗色兜底、选中文本高亮优化
- 把前台卡片背景色抽成站点设置项，前台文章正文、归档、分类、标签、作者卡片等区域统一跟随亮色 / 暗色变量
- 针对手机端和低性能设备，给前台背景与动效做降级，降低滚动卡顿和暗色背景残留问题

## 相比 v1.3.7 的变化

如果你当前线上或外部分发仍以 `v1.3.7` 为基线，那么 `v1.3.8` 相比 `v1.3.7` 可以重点关注下面这些变化：

### 1. 暗色主题不再只是“能看”，而是前后台真正形成统一设计

`v1.3.7` 已经把 Markdown 主题链路打通，但还残留几类明显的视觉问题：

- 后台管理壳子是蓝青色，编辑器 / 工具栏 / 预览面板却还是偏炭黑
- 后台预览区普通正文在暗色主题下依旧偏灰、偏暗，不够清晰
- 不同 Markdown 主题下，部分二级标题仍有“包裹一圈”的突兀感
- 私密文档右侧预览背景与后台整体主题没有融为一体

`v1.3.8` 这次做的是整套暗色视觉收口：

- 后台编辑器、工具栏、状态栏、预览区统一切到蓝青系暗色面板
- 后台预览正文、列表文本、引用、链接、标题层级全部重新提亮
- 站点 Markdown 主题增加统一 hotfix，按主题分别软化 `h2` 的外圈 / 包裹感，而不是强行做成一个模板
- 私密文档 / 预览容器和 Admin 整体色板重新对齐，减少割裂

这意味着：后台不再是“壳子一个颜色、内容一个颜色”，暗色模式整体会更像一个完整产品。

### 2. 代码块终于兼顾可读性、操作性和审美，不再只是能复制

此前代码块还存在几类典型问题：

- 长行代码没有足够友好的横向阅读与自动换行切换
- 展开代码逻辑容易被覆盖，看不到原先的折叠能力
- 暗色模式下未标注语言的 fenced code block 容易落成黑字
- 预览区和前台把“自动换行 / 展开代码”直接显示成文字，观感不够精致
- 代码被选中时的高亮颜色偏脏、偏硬，阅读体验不好

`v1.3.8` 把代码块交互重新整理为一条完整链路：

- 复制、自动换行、展开 / 收起统一升级为图标按钮
- 仍保留横向滚动能力，同时支持一键切到自动换行
- 代码折叠继续接入站点设置里的“代码块最大显示行数”，不再写死
- 未标注语言的代码块在暗色模式下补齐浅色文本兜底
- 编辑区与前台统一调整选中态、边框、容器层次和按钮风格

如果你的文章里经常出现命令块、环境变量行、配置片段或很长的代码示例，这版会比 `v1.3.7` 更顺手。

### 3. 前台卡片背景色正式可配置，暗色主题不再到处露出黑底

此前虽然已经开始把前台背景从纯黑调整出来，但还有两个体感问题：

- 有些区域铺满的是背景层，不是真正的内容卡片
- 手机和桌面上仍会看到局部黑底，没有完整跟随站点主题

`v1.3.8` 这次把前台表层背景进一步变量化：

- 站点设置新增前台卡片背景色（亮色 / 暗色）字段
- 前台文章正文、归档、标签、分类、导航、友链、搜索卡片、作者卡片等统一接入背景变量
- 页面 shell、正文容器、辅助卡片分层重新梳理，减少“正文是主题色、外围还是黑色”的情况
- 前台初始化逻辑和布局注入补齐对应变量同步

对站长来说，这意味着前台暗色风格终于不需要被固定在“黑底 + 少量主题点缀”，而是可以更完整地跟随后台想要的色彩方向。

### 4. 手机端与低性能设备的阅读流畅度更稳

这一轮还顺手解决了一个容易在真机上暴露的问题：

- 手机端打开前台文章后，下滑有时明显卡顿
- 暗色模式下某些背景在移动端仍保持固定黑底
- 雪花、连线、点击特效等视觉效果在移动端价值不高，却会额外吃掉性能

`v1.3.8` 的处理方式不是简单删掉特效，而是做条件降级：

- 对粗指针 / 触屏 / 小屏 / `prefers-reduced-motion` 场景自动关闭重动效
- 移动端背景和交互细节做更保守的降级处理
- 目录与文章页局部监听逻辑做减负，减少滚动压力

这让前台在手机端的实际浏览体验更稳定，也更接近真实线上场景需要的“顺滑优先”策略。

## 主要特性 / 修复

### Markdown / 主题 / 视觉系统

- 后台编辑器暗色主题从炭黑面板切换到蓝青系 Admin 风格，工具栏、状态栏、编辑区、预览区统一配色
- 新增前台卡片背景色亮色 / 暗色配置项，并通过站点设置链路下发到前台
- 前台与后台统一补齐 Markdown 主题 hotfix，重点修复多个主题在 `h2` 上的包裹感、描边感和过度装饰问题
- 私密文档预览区与普通编辑预览区统一暗色面板层次，避免局部纯黑背景突兀

### 代码块 / Mermaid / 阅读体验

- 代码块的复制、自动换行、展开 / 收起切换为图标按钮，预览区和前台交互更简洁
- 修复长代码块横向阅读体验，保留横向滚动并提供自动换行切换
- 修复暗色模式下未标注语言的代码块在预览区显示为黑字的问题
- 调整代码选中文本、代码块边框、按钮状态与折叠容器渐隐策略，减少“脏灰感”
- 继续保持 Mermaid 居中策略，并统一前后台容器边距与颜色兜底

### 前台布局 / 自适应 / 性能

- 前台文章页、归档页、分类页、标签页、友链页、导航页等卡片统一接入背景变量
- 文章正文铺满逻辑进一步修正，避免只铺满背景、不铺满内容层
- 首页“查看全部归档”在暗色模式下的文字颜色修复，避免黑字不可读
- 移动端与低性能设备自动关闭雪花、Canvas Nest、点击粒子、鼠标拖尾等高成本效果

### 后台站点设置 / 品牌配置

- 站点设置表单补齐前台卡片背景色字段
- 元数据写入链路增加颜色值校验与归一化，避免非法色值污染站点配置
- 前台布局头部同步链路补齐相关测试，确保暗色与亮色配置都能正确生效

## 部署前注意事项

发布 `v1.3.8` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.8`
- Git tag 使用 `v1.3.8`
- Docker Hub 推送使用本机 `pnpm release:images:push`
- 当前正式发布仍默认只推 `linux/amd64`

如果你的站点最近遇到下面任意现象，这版建议优先升级：

- 后台编辑器暗色模式下正文、列表、预览区对比不足，不容易看清
- 前台文章页、卡片页、手机端仍能看到明显黑底
- 长代码块没有优雅的换行 / 展开交互
- 某些 Markdown 主题的二级标题“包裹感”太重，显得突兀
- 手机端前台文章下滑明显卡顿

## 升级步骤

- 先备份当前生产环境的 `data/`、`log/`、`caddy/` 目录
- 把部署侧的镜像版本切到 `v1.3.8` 或对应不可变 tag
- 执行 `docker compose -f docker-compose.image.yml up -d`
- 升级后重点验证：
  - `/admin` 暗色编辑器与预览区是否已经切到统一蓝青系面板
  - 前台文章页、归档页、卡片页是否已经跟随站点背景变量，不再露出纯黑底
  - 长代码块是否能正常使用复制、自动换行、展开 / 收起
  - 手机端下滑文章页是否比旧版更顺畅
  - 暗色模式下“查看全部归档”等按钮文字是否正常可读

## 回滚方法

如果你在 `v1.3.8` 升级后需要快速回滚：

- 直接把 `docker-compose.image.yml` 或部署环境变量中的版本别名 tag 改回 `v1.3.7`
- 如果你需要严格锁定单次构建结果，则改回对应的不可变 tag
- 回滚后建议清理浏览器缓存并重新打开 `/admin` 与前台文章页，避免旧的主题 CSS、背景变量和缓存资源残留
