# v1.5.2

`v1.5.2` 是一次围绕后台暗色主题、ByteMD 编辑体验，以及前台/预览区 Markdown 深色视觉收口的补丁版本。

如果只用一句话概括：

> `v1.5.2` 把后台暗色模式从原来的蓝青调收口到更克制的石墨黑，同时保留编辑页需要的一点层次感，并把前台、代码块、Mermaid、私密文档预览一起统一到同一套暗色语义里。

## 本版摘要

这次版本主要集中在四件事：

- 默认 Markdown 编辑器继续保持 `ByteMD`
- 后台暗色主题改成更接近黑色的石墨系，不再默认走蓝青色
- 后台大多数卡片、表格、面板收口到同一底色，去掉多余边框和层次噪音
- 私密文档页、文章预览、前台代码块和 Mermaid 的暗色视觉统一到新的石墨黑体系

## 相比 v1.5.1 的变化

如果你已经在用 `v1.5.1`，这一版最直接的感受会是：

- 编辑器仍然是 ByteMD，不再回到 Milkdown
- 管理后台暗色模式不再默认偏蓝青
- 一些原来“层次很多”的卡片和预览区现在更干净
- 但文章 / 草稿 / 私密文档编辑页仍保留必要的边界，不会黑成一整片

这不是一次功能扩张版本，更像一次“视觉语言统一 + 主题配置补强”的补丁发布。

## 主要更新

### 1. ByteMD 继续作为默认编辑器

这一版没有再切换编辑器体系：

- 默认编辑器仍是 `ByteMD`
- 现有编辑页、草稿页、私密文档编辑流程不需要迁移
- 本地存储里的编辑器偏好逻辑保持兼容

### 2. 后台暗色主题改为石墨黑，而不是蓝青色

后台暗色主题这次从旧的蓝青基底，收口成更克制的石墨黑：

- 默认暗色背景从偏蓝的 `#111827` 升级到石墨黑 `#111315`
- 主强调色同步改成更低饱和的钢灰蓝，而不是高亮蓝青
- 服务端会自动把旧的内置蓝青暗色预设升级到新的石墨黑默认值

同时，这版还补了后台暗色预设能力：

- `炭黑石墨`
- `暖黑墨色`
- `冷黑金属`
- `高级自定义`

这样如果你不想全靠手填颜色，也可以直接选一套预设。

### 3. 编辑页去掉不必要的深色内层背景，但保留必要层次

此前后台暗色模式下，ByteMD 编辑区容易出现“外层一块、内层又一块”的深色叠层。

`v1.5.2` 对这部分做了收口：

- ByteMD 输入区内层背景改为透明继承，不再叠第二层深色底板
- 内部 `CodeMirror` / `CodeMirror-scroll` / `CodeMirror-lines` 不再单独绘制一块更深的背景
- 编辑器外层仍保留作为工作区边界的容器感
- 文章、草稿、私密文档编辑页保留必要边界，不会完全失去区块感

结果就是：

- 看起来更干净
- 仍然知道哪里是编辑区域
- 不会再出现蓝青 / 黑色 / 更深黑色叠三层的割裂感

### 4. 前台、预览、代码块、 Mermaid 一起切到石墨黑体系

这版不只改后台壳子，也把前台和预览相关的深色样式一起收口了：

- 前台暗色卡片默认底色改到 `#111315`
- Markdown 深色默认预设改成 `vanblog-graphite-dark-only.css`
- 前台代码块、后台预览代码块、文档代码块都从蓝青氛围改成石墨黑氛围
- Mermaid 深色展示也跟随同一套黑色基调，而不是继续偏蓝青

这样暗色模式下，前台文章、后台预览、代码块、流程图终于是一套语言，不会一半像石墨黑、一半像旧蓝青主题。

### 5. Admin 卡片、表格、面板统一底色，去掉丑边框

这一版另一个明显变化，是后台很多卡片和面板不再“每块一个色、每块一道边框”：

- 暗色模式下大多数卡片、ProCard、表格容器、描述面板统一使用同一表面底色
- 多余描边、阴影、悬浮叠层被显著减少
- 选中态和 hover 态尽量改成轻量表达，而不是再叠一个大色块

目标不是“完全没层次”，而是避免那种看起来像样式没统一好的假层次。

### 6. 私密文档预览区减少层次，空态文案融入背景

根据这次验收反馈，私密文档 `/admin/document` 做了进一步收口：

- 文档纯预览页减少多层嵌套背景
- `document-content-card` 内部 viewer 不再额外叠多层底色
- “请选择一个文档进行查看或编辑 / 在左侧文档树中创建文档库和文档” 这块空态文案不再放进突兀色块里
- 空态内容直接融入当前背景，只保留文字层级

这让私密文档页在暗色模式下更安静，也更接近“内容面板”而不是“卡片套卡片”。

## 配置与兼容性

这次版本虽然主要是视觉收口，但也补强了几项配置落点：

- 后台管理主题配置新增暗色预设映射
- 前台深色卡片底色支持独立配置 `frontCardBackgroundColorDark`
- 前台 Markdown 深色预设支持独立配置 `markdownDarkThemePreset`
- 旧版后台蓝青暗色默认值会自动迁移到新的石墨黑默认值

兼容性方面：

- 不新增环境变量
- 不需要数据库迁移
- 已有站点可直接从 `v1.5.1` 升级到 `v1.5.2`

## 发布与部署边界

`v1.5.2` 依旧保持当前的多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 仍然使用官方镜像
- AI 工作台继续包含在 `server` / `admin` 核心镜像中
- bundled FastGPT 仍然是可选叠加，不属于默认主栈公开拓扑

也就是说，这一版仍然是“主栈正常升级，AI 按需叠加”的模型。

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.5.2-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级建议

升级到 `v1.5.2` 后，建议重点看这几个地方：

1. `/admin/editor`、文章页、草稿页、私密文档编辑页是否仍使用 ByteMD
2. 暗色模式下编辑区是否已经去掉多余内层深色底板
3. 前台文章代码块和 Mermaid 是否已经从蓝青风格切到石墨黑风格
4. `/admin/document` 纯预览页是否已减少多层卡片叠加感
5. 空态文案区域是否已融入背景，而不是落在突兀色块上

如遇到旧 CSS 缓存，建议强刷浏览器资源再确认。

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.1-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只是样式看起来没有变化，优先排查浏览器缓存和 CDN 缓存，而不是直接判断镜像回滚失败。
