# v1.3.11

`v1.3.11` 是在 `v1.3.10` 基础上，围绕 Mermaid 图导出、前台 Mermaid 渲染兜底、代码块顶部留白收紧，以及后台登录页和文档预览细节收口做的一次正式补丁发布。

如果只用一句话概括这个版本：

> `v1.3.11` 重点解决“Mermaid 图前台偶发不渲染、匿名访问和局域网访问看到的表现不一致、代码块首行离顶部过远、Mermaid 图不能直接导出”这一批已经影响实际使用和验收的问题。

## 本版摘要

本次版本主要集中在四个方向：

- 为前台文章页、后台编辑器预览和后台文档查看器里的 Mermaid 图统一补上右上角 `PNG / SVG` 导出工具
- 前台文章页补充 Mermaid 渲染兜底逻辑；即使 `@bytemd/plugin-mermaid` 的异步替换不稳定，也会把原始 `language-mermaid` 代码块渲染成 SVG
- 代码块共享样式继续收口，把工具条与首行代码之间的顶部空白压缩到更紧凑的阅读间距，同时保留之前已经缩小过的语言标签与图标尺寸
- 补齐缺失文章的 404 返回、局域网开发来源白名单，以及后台登录页片卡宽度收口，减少“后台预览正常但前台异常”或“局域网看起来和本机不一致”的排查成本

## 相比 v1.3.10 的变化

如果你当前线上或镜像分发仍以 `v1.3.10` 为基线，那么 `v1.3.11` 相比 `v1.3.10` 建议重点关注下面这些变化：

### 1. Mermaid 图现在可以直接导出 PNG / SVG

这一版最直观的变化，是 Mermaid 图右上角新增了两个轻量工具按钮：

- 导出 PNG
- 导出 SVG

这条链路同时覆盖：

- 前台文章页
- 后台编辑器预览区
- 后台文档查看器

导出实现直接基于已经渲染出的 Mermaid SVG：

- `SVG` 走原始标记导出，便于二次编辑或继续嵌入别处
- `PNG` 在本地浏览器内栅格化，自动继承当前图表所在容器的背景色

对实际使用来说，这意味着你不需要再手工截图 Mermaid 图，也不需要切到第三方站点重新贴源码导出。

### 2. 前台 Mermaid 渲染补了兜底，不再只依赖插件异步替换成功

`v1.3.10` 已经把 Mermaid 的明暗主题统一到了官方 light / dark 配置，但前台仍可能出现一种让人很难排查的场景：

- 后台预览正常
- 前台文章页偶发只看到原始 Mermaid 代码块
- 匿名访问、局域网 IP 访问或某些开发态访问方式下表现还不完全一致

`v1.3.11` 的做法是继续往下补一层：

- 直接扫描前台 Markdown 区域里的 `pre > code.language-mermaid`
- 如果 Mermaid 代码块还没被替换成图表，就在 viewer effect 里主动调用 Mermaid 渲染并替换为 SVG
- 通过 `MutationObserver` 继续兜住后续异步插入的 Mermaid 代码块

这样处理后，前台即使碰到插件替换时序不稳定，也能把 Mermaid 图实际渲染出来，而不是把原始代码直接暴露给读者。

### 3. 代码块顶部留白继续收紧，首行代码更贴近工具条

之前已经把代码块右上角语言标签和工具图标缩小到了更满意的尺寸，但顶部仍然存在“第一行代码离块顶太远”的问题。

`v1.3.11` 继续把这块共享样式收口：

- 缩小代码块外层 wrapper 的顶部 padding
- 把右上角工具条位置上移
- 把代码内容区的 `padding-top` 从较大的预留值压到更紧凑的高度
- 同步前台亮色 / 暗色、后台亮色 / 暗色以及 Markdown theme hotfix 的同一套数值

最终效果是：第一行代码与顶部工具条之间的空白明显缩小，视觉上更像同一套完整的代码面板，而不是“工具条浮在上面，正文又另起一块”。

### 4. 前台路由和后台登录页的边缘问题也一起收口了

除了 Mermaid 和代码块本体，这次还顺手补了几处会影响验收的边缘问题：

- 缺失文章现在会明确返回 `404`，而不是继续渲染一个只有空壳数据的文章页
- Next 开发态 `allowedDevOrigins` 会自动纳入局域网 IPv4 地址和配置里的允许域名，减少通过局域网 IP 调试时资源 / HMR 被拦的情况
- 后台登录页片卡宽度收口为固定上限，不再在某些布局下接近整屏宽度
- 后台文档查看器复用 Mermaid 导出工具和更紧凑的按钮样式，避免预览区和前台表现继续分叉

## 主要特性 / 修复

### Mermaid / Markdown 呈现

- 前台文章页新增 Mermaid 渲染兜底，原始 `language-mermaid` 代码块会在 viewer effect 中主动转成 SVG
- Mermaid 图新增 `PNG` / `SVG` 导出工具，前台文章、后台编辑器预览、后台文档查看器保持一致
- Mermaid 工具条以更轻量的小尺寸图标呈现，不抢正文注意力
- Mermaid 容器改为相对定位，统一承接导出工具条定位
- 补充 Mermaid 导出与前台 Mermaid 兜底渲染相关测试

### 代码块阅读体验

- 统一收紧前台与后台亮 / 暗主题代码块的顶部间距
- Markdown theme hotfix 同步到相同的代码块 spacing 数值，避免“主题 CSS 和默认 CSS 不一致”
- 保持之前已经缩小后的语言标签 / 工具图标尺寸不变，只减少它们与第一行代码之间的空白

### 前台路由与局域网调试稳定性

- 缺失文章页改为明确 `notFound()` / `notFound: true` 返回，避免前台继续渲染空文章壳
- `getPostPagesProps` 对无效文章数据返回 `article: null`，减少前后台出现 `article/0/*` 这类错误请求
- Next 开发态允许来源自动吸收机器上的局域网 IPv4 地址，降低通过局域网 IP 验证时的跨来源开发限制

### 后台界面细节

- 后台登录页片卡增加最大宽度和自动居中，不再异常拉宽
- 后台文档查看器同步 Mermaid 导出工具样式与交互
- 后台代码块共享样式与前台保持一致，阅读体验更统一

## 部署前注意事项

发布 `v1.3.11` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.11`
- Git tag 使用 `v1.3.11`
- Docker Hub 推送仍以本机 `pnpm release:images:push` 为准
- 当前正式发布仍默认只推 `linux/amd64`

如果你的站点最近遇到下面任意现象，这版建议优先升级：

- Mermaid 图在后台预览正常，但前台文章页偶发只显示原始代码块
- 想直接从文章页或后台预览区导出 Mermaid 图给别人使用
- 代码块工具条已经满意，但首行代码离顶部仍然显得太空
- 通过局域网 IP 或匿名访问验证前台时，效果与本机访问不一致
- 某些旧文章路径实际已经不存在，但前台还会渲染一个空壳页面

## 升级步骤

如果你当前运行的是 `v1.3.9`：

1. 备份当前 `data/`、`caddy/`、`log/` 目录
2. 将发布镜像标签切到 `v1.3.11`
3. 执行：

```bash
docker compose -f docker-compose.image.yml pull
docker compose -f docker-compose.image.yml up -d
```

如果你使用不可变 tag：

```bash
export VANBLOG_DOCKER_REPO=kevinchina/deeplearning
export VANBLOG_RELEASE_SUFFIX=v1.3.11-<image-id>
docker compose -f docker-compose.image.yml up -d
```

## 回滚方法

如果升级后需要回滚，直接把 `VANBLOG_RELEASE_SUFFIX` 改回上一版即可：

```bash
export VANBLOG_DOCKER_REPO=kevinchina/deeplearning
export VANBLOG_RELEASE_SUFFIX=v1.3.9-<image-id>
docker compose -f docker-compose.image.yml up -d
```

如果你使用的是版本别名，也可以临时改回：

```bash
export VANBLOG_RELEASE_SUFFIX=v1.3.9
docker compose -f docker-compose.image.yml up -d
```
