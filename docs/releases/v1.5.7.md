# v1.5.7

`v1.5.7` 是一次围绕后台文章编辑页预览区显示问题的补丁发布，重点是把 ByteMD 在较窄管理区宽度下被隐藏的预览区恢复出来。

如果只用一句话概括：

> `v1.5.7` 保留 ByteMD 的分栏预览模式，但在较窄后台内容宽度下自动改为上下堆叠布局，避免 `/admin/editor` 中预览区直接消失。

## 本版摘要

这一版主要集中在三件事：

- 修复后台文章编辑页在较窄内容宽度下看不到预览区的问题
- 保持 ByteMD 使用原生 `split` 模式，避免引入额外预览渲染链路带来的运行时回归
- 补充针对窄屏编辑布局的管理端烟雾回归验证

## 相比 v1.5.6 的变化

如果你已经在使用 `v1.5.6`，这一版最直接的变化会出现在后台文章编辑页：

- `/admin/editor?type=article&id=...` 在较窄的后台内容区域里，预览区不会再直接消失
- ByteMD 会在宽度不足时从左右分栏切换成上下堆叠，继续同时展示编辑区和预览区
- 管理端回归测试现在会专门检查窄宽度下的预览区可见性和内容渲染

这不是一次新功能版本，而是一次针对后台编辑体验退化的定点修复。

## 主要更新

### 1. 修复 ByteMD 窄宽度下的预览区隐藏

这次问题的根因不在文章内容本身，而在后台内容区宽度收窄后，ByteMD 的布局没有继续给预览区留下可见空间：

- 编辑器仍处于双栏预览语义
- 但在较窄的管理区内容宽度下，预览区会因为布局收缩而表现为“看不到”
- 实际结果就是后台看起来只剩编辑区，用户无法确认渲染结果

`v1.5.7` 现在在较窄宽度下把 ByteMD 的编辑区和预览区改为上下堆叠，继续保留两块面板：

- 编辑区占满上半部分
- 预览区占满下半部分
- 两块面板都保持 `width: 100%`，不再因为横向挤压而消失

### 2. 继续使用 ByteMD 原生 `split` 模式

修复过程中评估过把预览区完全替换成自定义 `DocumentViewer` 渲染链路，但这会把 ByteMD 的预览生命周期改复杂，并引入额外运行时风险。

这一版最终选择更保守也更稳定的方案：

- 保留 ByteMD 原生 `mode="split"`
- 不额外接管 `overridePreview`
- 只在布局层处理窄宽度下的显示方式

这样可以把修复范围限制在界面布局，不把本次补丁扩大成编辑器渲染架构调整。

### 3. 补充窄屏预览回归测试

为了避免后续再次把这个问题带回来，这一版补了一条管理端烟雾回归检查，专门约束：

- 在 `1100px` 左右的后台视口宽度下，ByteMD 预览区必须可见
- 预览区必须仍然渲染出 markdown 内容
- 编辑区和预览区必须改为上下堆叠，而不是继续横向挤压

这类检查虽然是 UI 层面的，但对这次问题的回归保护很直接。

## 配置与兼容性

`v1.5.7` 不引入新的环境变量，也不需要数据库迁移：

- 不新增 `.env` 配置项
- 不需要数据库结构迁移
- 不改变 AI / FastGPT 部署方式
- 从 `v1.5.6` 可直接升级

## 发布与部署边界

`v1.5.7` 继续保持当前多镜像发布边界：

- 核心发布物仍然只有 `caddy`、`server`、`website`、`admin`、`waline`
- `postgres` 与 `redis` 继续使用官方镜像
- AI 工作台继续随 `server` / `admin` 核心镜像发布
- bundled FastGPT 仍然是可选叠加，不属于默认主栈公开拓扑

## docker compose 配置单

正式部署仍推荐：

- `docker-compose.image.yml`
- `.env.release.example`

示例环境变量：

```env
VANBLOG_DOCKER_REPO=kevinchina/deeplearning
VANBLOG_RELEASE_SUFFIX=v1.5.7-<image-id>
```

主栈部署：

```bash
docker compose -f docker-compose.image.yml up -d
```

连接已有私有 FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml up -d
```

同时启用 bundled FastGPT：

```bash
docker compose -f docker-compose.image.yml -f docker-compose.ai-qa.yml -f docker-compose.fastgpt.yml up -d
```

如果只是 latest 快速体验：

```bash
docker compose -f docker-compose.latest.ai.yml up -d
```

## 升级建议

升级到 `v1.5.7` 后，建议重点看这几个地方：

1. 后台文章编辑页在普通桌面宽度和较窄内容宽度下，是否都还能同时看到编辑区与预览区
2. ByteMD 在较窄布局下，是否已经改成上下堆叠，而不是继续横向压缩
3. 预览区中的 Markdown、代码块、Mermaid、数学公式是否仍然正常渲染
4. 后台切换暗色主题、切换编辑器、刷新页面后，预览区是否仍然可见

## 回滚方法

如果线上想回滚到上一版，只需要把：

```env
VANBLOG_RELEASE_SUFFIX=v1.5.6-<image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```

如果只有后台浏览器表现异常，优先强刷管理端静态资源缓存后再确认。
