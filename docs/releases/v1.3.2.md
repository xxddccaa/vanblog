# v1.3.2

`v1.3.2` 是在 `v1.3.0` 的 Node 24 与多镜像基线上继续打磨的一次稳定版发布。

如果只用一句话概括这个版本：

> `v1.3.2` 重点补齐了 `/admin` 在真实使用中的一批问题：初始化/登录/找回密码暗色主题适配、编辑器正文加载与缓存覆盖、Safari 等浏览器下的编辑器布局，以及后台静态资源缓存策略。

## 本版摘要

本次版本的重点集中在三个方向：

- 修复后台 `/admin` 在真实博客数据上的编辑器异常
- 补齐后台认证页与编辑器的暗色主题细节
- 收紧正式发布流程，确保 GitHub / Docker Hub / 版本说明 / compose 配置单保持一致

## 相比 v1.3.0 的变化

如果你当前线上或外部分发仍然以 `v1.3.0` 为基线，那么 `v1.3.2` 相比 `v1.3.0` 可以重点关注下面这些变化：

### 1. `/admin` 编辑器可用性明显增强

`v1.3.0` 已经完成 Node 24 升级、宿主机调试链路和一批后台兼容修复，但后台编辑器在真实博客内容、浏览器缓存、公网转发 origin 下仍存在一些使用门槛。

`v1.3.2` 继续补齐了这些问题：

- 编辑页与代码页改为从 `location.search` 读取参数，避免 `history.location.query` 在当前运行时下不稳定
- 修复文章编辑器中“前台有正文、后台进入编辑页却看不到内容”的问题
- 修复本地空白缓存覆盖服务端正文的问题
- 修复编辑器高度链断裂后正文区域被压成 `0` 高度的问题
- 优化公网转发地址与本机地址分属不同 origin 时的排查与调试体验

### 2. 暗色主题适配从“能用”提升到“可长期使用”

相比 `v1.3.0`，`v1.3.2` 进一步补齐了后台暗色主题的细节，包括：

- `/admin/init`
- `/admin/user/login`
- `/admin/user/restore`
- `/admin/editor`

这次不仅修复了白底信息块，还补齐了：

- 编辑器工具栏
- 左侧 Markdown 编辑区
- 右侧预览区
- 状态栏
- 工具悬浮提示层
- 暗色模式下的图标颜色继承

对于长期使用暗色后台的博客，这一版比 `v1.3.0` 更值得升级。

### 3. 后台静态资源缓存策略更适合持续发布

`v1.3.2` 调整了后台静态资源的缓存行为：

- `/admin/index.html` 强制重新校验，避免浏览器长期吃旧壳文件
- 带 hash 的静态资源允许长期缓存

这能降低“代码已更新但用户浏览器仍在加载旧后台壳文件”的概率，尤其适合多次迭代发布后的线上环境。

### 4. 发布流程进一步收敛到本机

`v1.3.2` 明确收紧了正式发布流程：

- GitHub 上不再保留会误触发镜像构建、测试环境推送、ARM 测试镜像推送或文档镜像部署的 workflow
- 正式发布统一以本机命令为准
- 每个版本都要求同步维护版本说明与 compose 配置单

这意味着相比 `v1.3.0`，`v1.3.2` 的发布纪律更清晰，减少了 push/tag 后意外触发远端 Action 的噪音。

## 主要特性 / 修复

### 后台认证与初始化页

- 新增 `AuthThemeSync`，让初始化、登录、找回密码等不走主后台 layout 的页面也能同步主题
- 修复暗色模式下提示块、表单、卡片、背景和文本颜色不一致的问题

### 后台编辑器

- 修复编辑器参数读取逻辑
- 修复缓存空白内容覆盖服务端正文
- 修复 Safari / 公网访问场景下编辑器正文不显示
- 修复暗色主题下编辑器细节，包括工具栏、提示层和图标

### 调试与发布文档

- `AGENTS.md` 增补了 Docker 全量重启建议
- `AGENTS.md` 和 `docs/host-debug.md` 补充了 admin debug token / MCP 调试边界与建议
- `RELEASE.md` 补充了 Git tag、版本 wiki / release note、compose 配置单要求

## 部署前注意事项

发布 `v1.3.2` 前，请确认：

- 宿主机 Node 版本满足 `>=24 <25`
- 根目录 `package.json` 版本为 `1.3.2`
- Git tag 使用 `v1.3.2`
- Docker Hub 推送使用本机 `pnpm release:images:push`
- 生产部署优先使用不可变镜像标签，而不是 `latest`

如果你的使用场景依赖后台编辑器，请在升级后优先验证：

- `/admin/article`
- `/admin/editor?type=article&id=<id>`
- `/admin/user/login`
- `/admin/init`

## 升级步骤

从旧版本升级到 `v1.3.2`，推荐步骤：

```bash
docker compose -f docker-compose.image.yml pull
docker compose -f docker-compose.image.yml up -d
```

如果你使用的是源码构建部署，则执行：

```bash
docker compose down
docker compose up -d --build
```

## 回滚方法

如果 `v1.3.2` 不符合预期，把部署使用的版本后缀切回上一版即可，例如：

```env
VANBLOG_RELEASE_SUFFIX=v1.3.0-<old-image-id>
```

然后重新执行：

```bash
docker compose -f docker-compose.image.yml up -d
```
