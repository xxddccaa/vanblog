# v1.1.0

`v1.1.0` 是数据库架构升级版本：主站从 MongoDB 迁移到 PostgreSQL + Redis。

这个版本的推荐迁移方式不是“直接搬旧 MongoDB 数据目录”，而是：

1. 先用 `v1.0.3` 导出旧站点 JSON
2. 再部署 `v1.1.0`
3. 在新后台导入 JSON 完成迁移

## 核心变化

- 主业务数据从 MongoDB 切换到 PostgreSQL
- 缓存与任务状态使用 Redis
- 不引入 Elasticsearch，优先保证轻量和部署成本
- 导入恢复支持更多站点数据，补齐私密文档、动态、思维导图等内容
- 导入恢复改为异步任务，可查看进度
- 支持“跳过统计数据导入”，避免导入无意义的访问明细
- 初始化页现在只强制要求“登录用户名”和“登录密码”，更适合先初始化再导入备份
- 默认部署模式改为 HTTP-only；如果你需要内置 Caddy HTTPS，可叠加 `docker-compose.https.yml`

## 部署方式

推荐直接使用仓库里的：

- `docker-compose.image.yml`
- `.env.release.example`
- 可选：`docker-compose.https.yml`

最小流程：

```bash
cp .env.release.example .env
# 按需修改 .env 里的端口、目录、密码、邮箱
docker compose -f docker-compose.image.yml pull
docker compose -f docker-compose.image.yml up -d
```

初始化入口：

```text
http://<你的域名或IP>/admin/init
```

如果你只是为了导入旧站备份，那么初始化页里只要先填：

- 登录用户名
- 登录密码

其余站点信息都可以留空，导入 JSON 时会覆盖。

## 新版运行时依赖

- `postgres:16-alpine`
- `redis:7-alpine`

注意：

- 不要把 `postgres`、`redis` 暴露到公网
- 生产环境入口仍然应该只经过 Caddy
- 如果你想固定某次发布，请优先使用不可变 tag：`v1.1.0-<image-id>`

## 迁移建议

推荐顺序：

1. 在旧系统用 `v1.0.3` 导出 JSON
2. 在新系统完成初始化
3. 导入旧 JSON
4. 检查文章、私密文档、动态、思维导图、分类、标签、独立页面
5. 检查站点设置、菜单、友链、关于页等配置
6. 验证前台访问与后台登录是否正常

## 兼容性说明

- `v1.1.0` 的目标是兼容旧系统导出的 JSON
- 不承诺直接兼容旧版 MongoDB 数据目录原地启动
- 如果你要做迁移，请把“导出 JSON -> 导入 JSON”作为标准路径
